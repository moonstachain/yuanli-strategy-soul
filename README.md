# 原力战略三部曲 · Soul Operating Base

> **一条公理，三部十二模块，一个灵魂地基仓。**
>
> 本仓是「原力战略三部曲」未来所有衍生任务的**唯一入口仓**与**基石概念体系**：承接概念宪法、龙骨总纲、十二模块、术语词典、跨部接口、模板工具、Codex 执行、项目看板、知识库治理与递归回写。

---

## 0. 仓库定位

本仓不是正文仓，不替代：

- `yuanli-strategy-trilogy`：三部曲内容工程工作仓 / 书稿与课程孵化仓 / 外显表达仓
- `yuanli-strategy-trilogy-private`：三部曲私有知识与骨架仓
- 未来各产品 / 网站 / 课程 / Agent 仓

本仓的定位是：

> **原力战略三部曲的 Soul / Operating Base**  
> 用来保证所有衍生任务不偏离母公理、不重造轮子、不丢失上下文、不把一次性工作误当成系统工程。

未来所有项目遵循：

```text
所有项目从 soul 出发，所有经验回到 soul。
```

---

## 1. 入口仓制度

从现在开始，所有原力战略三部曲的衍生任务，默认先进入本仓：

```text
新想法 / 新需求 / 新项目
→ 先在 yuanli-strategy-soul 开 Issue
→ 归属裁决：资产 / 创业 / OS / 跨三部 / 治理
→ 模块裁决：A1-C4
→ 脊骨裁决：ʸx / 你×AI / 借势×合力 / 信息五层 / OSA / 递归回写
→ 拆解任务
→ 分发到内容仓 / 网站仓 / Codex 执行仓 / Agent 仓
→ 完成后回写本仓
```

### 1.1 不再直接开工

任何新任务不要直接丢给 Codex、不要直接进入具体仓、不要直接写页面。先在本仓开 Issue，完成：

```text
归属 → 边界 → 输出物 → 验收 → 回写
```

### 1.2 本仓是“任务中枢”，不是“所有代码堆积地”

本仓负责：

```text
定义任务
裁决归属
沉淀模板
记录路线图
回写知识
分发执行
```

具体执行仍可分发给：

```text
内容仓 / 网站仓 / 应用仓 / Agent 仓 / Codex 工作仓
```

## 2. 母公理

整套体系只有一条根：

> **ʸx：做会自我繁殖的事，不做一次性的事。**

所有任务启动前，都必须问：

```text
这个任务完成后，是否会让下一次任务更容易、更高级、更可复用？
```

如果不能，它只是一次性劳动；如果能，它才属于原力战略三部曲的体系内任务。

任何新任务，先回答五问：

```text
1. 它属于原力资产、原力创业、原力OS，还是跨三部 / 治理？
2. 它服务哪条母公理或共享脊骨？
3. 它归属 A1-C4 哪个模块？
4. 它的输入与输出物是什么？
5. 它如何递归回写，成为下一次任务的资产？
```
