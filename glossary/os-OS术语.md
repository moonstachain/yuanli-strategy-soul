## 0. C 轴边界

```text
《原力 OS》做深：一纸文脉、一个大脑、一张地图、一条链路。
《原力 OS》不重讲完整自我觉醒，不重讲完整创业四关。
它把资产与创业的输出写入系统，让每次任务成为下一次能力。
```

## 1. 术语表

| 战略边界清单 | 不做清单 | C1 | 服务谁、不服务谁、做什么、不做什么的边界清单。 | 不是价值观口号。 | C1 |
| 风格词典 | 像我说话的规则 | C1 | 概念、隐喻、语气、审美、表达节奏的稳定规则。 | 不是文案模板。 | C1 |
| 一个大脑 | 记忆系统 | C2 | 把知识、案例、术语、样例、数据、任务结果沉淀成可检索、可调用、可版本化的系统。 | 不是资料堆。 | C2 |
| 知识索引 | 找得到知识 | C2 | 让人和 AI 知道知识在哪里、是否可信、如何调用。 | 不是目录堆砌。 | C2 |
| 案例索引 | 找得到案例 | C2 | 记录项目、客户、交付、成功、失败和复用方式。 | 不是故事库。 | C2 |

| 一张地图 | 判断系统 | C3 | 把文脉、大脑、市场验证和创业结果组织成可取舍、可判断、可行动的战略地图。 | 不是思维导图。 | C3 |

| 行动阈值 | 何时推进/暂停/放弃 | C3 | 预先定义触发行动的判断条件。 | 不是模糊感觉。 | C3/C4 |
| 一条链路 | 手脚 / 行动系统 | C4 | 把战略判断转成可执行、可验收、可回写、可自动化的行动流程。 | 不是待办事项。 | C4 |
| OSA | 决策三段 | C4 | Objective、Strategy、Action 的任务拆解链路。 | 不是直接行动清单。 | C4 |
| 最小闭环 | goal-action-evidence-writeback-mirror | C4 | 最小真实任务从目标到证据、写回、镜像同步的闭环。 | 不是做完一件事。 | C4 |

| Writeback | 写回 | C4 | 把任务产物回写成知识、模板、案例、skill 或地图更新。 | 不是归档。 | C4 |
| Mirror | 镜像同步 | C4 | 把写回结果同步到系统可调用的记忆与上下文中。 | 不是复述。 | C4 |

| Skill 蒸馏卡 | case → pattern → skill | C4 | 从个案中提炼可复用技能的卡片。 | 不是案例摘抄。 | C4 |

## 2. 禁止混用

```text
C1 不替 A1-A3 重做自我觉醒。
C2 不变成资料仓。
C3 不变成思维导图。
C4 不变成待办清单。
OS 四件套必须合在一起，不能只做工具或自动化。
```
