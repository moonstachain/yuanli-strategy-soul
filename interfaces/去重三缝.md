## 0. 总裁决

```text
重叠内容不靠“每本都讲一点”解决。
唯一正确动作：一处做深，余处只留引用。
```

三部曲是一个系统，不是三本独立散书。每条缝必须明确：

```text
谁做深？
谁只引用？
引用到哪里？
接口输出是什么？
下游接什么？
```

## 1. 缝一：觉醒 / 找你 / 天才区

### 冲突内容

```text
找到你
天才区
人格资产
三圈交集
原力资产
母体发现
AI 时代你凭什么不可替代
```

### 做深归属

```text
归《原力资产》做深。
对应模块：A1 发现母体、A2 回到母体、A3 获得原力、A4 显化原力。
```

### 只留引用

```text
《原力创业》关一不再叫“原力觉醒”，改为“原力借势”。
创业关一只引用 A 轴已经形成的“合力 / 原力资产 / 创业输入包”。
```

### 接缝条文

```text
觉醒（找你）整本迁入《原力资产》。
创业 B1 不再重讲“我是谁”，只回答“这个时代为什么现在需要我”。
合力侧来自 A4 创业输入包，借势侧由 B1 做深。
```

### 标准引用句

```text
关于“我是谁 / 我的原力是什么 / 合力如何形成”，详见《原力资产》A1-A4；本关只处理“带着已经显化的原力，如何进入时代势能”。
```

### 输出接口

```text
A4 输出：创业输入包
B1 接收：创业输入包 + 市场验证记录 + 天选用户线索 + 贵问题线索
```

### 禁止事项

```text
B1 禁止重新展开三圈交集。
B1 禁止重新展开天才区测评。
B1 禁止把原力资产当作创业关一内部产物。
B1 可以引用“合力”，但必须指向 A 轴。
```

## 2. 缝二：觉醒 vs 变现

### 冲突内容

```text
天选用户
心理账户
品类创新
贵任务
贝叶斯创业
非线性回报
用户愿意付费的价值表达
```

### 做深归属

```text
变现深内容归《原力创业》做深。
对应模块：B2 品类独创、B3 模式升维、B4 壁垒锁定。
```

### 只留引用

```text
《原力资产》只讲到 A4 显化原力：天选用户线索、贵问题线索、心理账户线索、市场验证记录。
资产书不展开完整品类、定价、商业模式、壁垒。
```

### 接缝条文

```text
《原力资产》止于“原力是否被世界需要”的验证。
《原力创业》接住这份验证，把它变成品类、模式、壁垒和财富飞轮。
```

### 标准引用句

```text
当你按下确定键、出现天选用户和贵问题线索后，完整变现路径进入《原力创业》：品类独创、模式升维、壁垒锁定。
```

### 输出接口

```text
A4 输出：天选用户线索 / 贵问题线索 / 心理账户线索 / 市场验证记录 / 创业输入包
B2-B4 接收：用户语言、价值主张、心理账户定位、三链路、利润容器、壁垒地图
```

### 禁止事项

```text
A4 禁止展开完整品类战略。
A4 禁止展开产品阶梯和商业模式。
A4 禁止展开 16 控制点。
A4 可以留下线索，但必须把深内容交给 B 轴。
```

## 3. 缝三：元 AI / 自用元 AI / 业务班底

### 冲突内容

```text
元 AI
我的说明书
AI 镜像
一纸文脉
一个大脑
一张地图
一条链路
AI 班底
agent workflow
```

### 做深归属

```text
自用元 AI：归《原力资产》做深。
业务元 AI / 造物班底：归《原力 OS》做深。
```

### 拆分裁决

| 内容 | 做深归属 | 说明 |
|---|---|---|
| 为自己造元 AI | A3 获得原力 / C1 接口 | AI 作为镜子、放大器、预言家，帮助我看见母体、外化能力。 |
| 我的说明书 V3 | A3/C1 | A3 形成自我说明书，C1 把它正典化为一纸文脉。 |
| 一纸文脉 | C1 | 进入 OS 四件套，成为 AI 与团队可调用的最小正典上下文。 |
| 一个大脑 | C2 | 记住知识、案例、术语、样例和数据。 |
| 一张地图 | C3 | 做战略判断、能力配置、优先级和行动路径。 |
| 一条链路 | C4 | 把判断变成可执行、可验收、可回写的行动流程。 |
| AI 班底 / agent workflow | C4 | 属于业务执行系统，不属于资产书自我认知部分。 |

### 接缝条文

```text
“元 AI”一词必须拆成两层：
1. 为自己造的元 AI：服务自我认知、母体照见、能力外化，归 A 轴。
2. 为业务造的元 AI 班底：服务系统交付、自动化、回写和造物，归 C 轴。
```

### 标准引用句

```text
本章只处理“为我自己造元 AI”：让 AI 更懂我的母体、判断和表达。至于把元 AI 造成业务班底、知识大脑、战略地图和自动化链路，详见《原力 OS》C1-C4。
```

或：

```text
本章处理原力 OS 四件套：文脉、大脑、地图、链路。关于“我是谁”以及我的说明书如何形成，详见《原力资产》A1-A3。
```

### 输出接口

```text
A3 输出：我的说明书 V3 / 原力能力包 / AI 外化样例库
C1 接收：我的说明书 V3 / 母体定义 / 品味函数 / 长期承诺 / 战略边界
C2-C4 接收：文脉约束、知识资产、地图判断、行动链路
```

### 禁止事项

```text
A3 禁止展开完整 OS 四件套。
C1-C4 禁止重新讲完整自我觉醒。
C4 禁止把 agent workflow 当成“学 AI 工具”。
元 AI 不得无归属地泛用。
```

## 5. 审查规则

任何新章节、新概念、新模板、新页面、新 Codex 任务，必须先回答：

```text
1. 它是否触碰三缝之一？
2. 如果触碰，谁做深？谁只引用？
3. 它的标准引用句是什么？
4. 它的输出接口是什么？
5. 它的回写位置在哪里？
```

不能回答以上问题，不进入正典层。
