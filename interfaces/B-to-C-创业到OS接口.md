# B-to-C · 创业到 OS 接口 · v0.1

> Issue: #206  
> 归属：interfaces / B→C 接口  
> 上游：B4 壁垒锁定  
> 下游：C1-C4 原力 OS  
> 目标：把《原力创业》的生意结果、壁垒结果和飞轮经验，转译成《原力 OS》的文脉、大脑、地图、链路输入。

---

## 0. 接口裁决

```text
B 轴负责跑出生意：借势、品类、模式、壁垒。
C 轴负责把生意写成系统：文脉、大脑、地图、链路。
```

B→C 的核心不是“复盘创业”，而是：

```text
这次创业结果，如何让下一次判断、交付、增长、自动化更强？
```

## 2. 下游拆分

| B4 输出 | C 轴接收 | 说明 |
|---|---|---|
| 品类定位与价值主张 | C1 一纸文脉 | 更新战略边界、服务对象、语言风格 |
| 用户语言与案例 | C2 一个大脑 | 写入用户语言库、案例库、样例库 |
| 三链路与壁垒地图 | C3 一张地图 | 变成事业系统地图、能力组件地图、控制点地图 |
| 交付 SOP 与增长动作 | C4 一条链路 | 变成流程、agent workflow、Codex task |
| 飞轮经验 | C1-C4 | 同时更新文脉、记忆、判断和行动 |

## 4. 验收标准

B→C 接口只有满足以下条件，才算完成：

```text
1. 至少有一个可回写到 C1 的战略边界或表达更新。
2. 至少有一个可回写到 C2 的案例、术语、样例或用户语言。
3. 至少有一个可回写到 C3 的控制点、瓶颈或地图更新。
4. 至少有一个可回写到 C4 的 SOP、流程、agent 或 Codex task。
5. 能说明本轮创业如何让下一轮更强。
```

## 5. 禁止事项

B4 禁止停留在“我们有壁垒”。
