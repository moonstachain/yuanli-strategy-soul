# C-to-A · OS 回写资产接口 · v0.1

> Issue: #206  
> 归属：interfaces / C→A 接口  
> 上游：C4 一条链路  
> 下游：A1/A2/A3/A4 原力资产升级  
> 目标：把 OS 执行、任务闭环、案例沉淀、自动化经验回写到原力母体，让下一轮“我是谁”更清楚、更强、更可复利。

---

## 0. 接口裁决

```text
C 轴不是终点。
OS 的终点是让母体升级。
```

C→A 的核心不是“系统做完了”，而是：

```text
这次系统执行，让我对自己的母体、能力、边界、用户和长期承诺更清楚了什么？
```

## 2. 下游回写

| C4 输出 | A 轴回写 | 说明 |
|---|---|---|
| 任务证据 | A1 发现母体 | 哪些任务最能显现真实母体 |
| Evolution Note | A2 回到母体 | 哪些边界、品味、注意力需要更新 |
| Skill 蒸馏卡 | A3 获得原力 | 哪些能力已经可调用、可复用、可训练 |
| 用户反馈与系统数据 | A4 显化原力 | 哪些用户、问题、市场信号被验证 |
| 自动化结果 | A2/A3 | 哪些算力活可以外包，哪些原力活必须保留 |

## 4. 验收标准

C→A 接口只有满足以下条件，才算完成：

```text
1. 不只是复盘任务，而是明确更新母体、能力、边界或用户判断。
2. 至少回写 A1-A4 中一个模块。
3. 至少产生一个可复用资产：模板、skill、案例、判断、样例或新任务。
4. 能说明下一轮从哪里重新开始。
5. 能让下一轮“发现母体 / 回到母体 / 获得原力 / 显化原力”更强。
```

---

## 5. 禁止事项

```text
C4 禁止报“任务完成”而不回写。
C4 禁止只写工作日志。
C4 禁止把自动化结果当成最终成果。
A 轴禁止忽略来自系统执行的新证据。
```
