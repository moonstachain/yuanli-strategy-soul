# Module Asset Pack Standard · 原力战略三部曲

Date: 2026-06-30
Status: seed standard

> 本标准用于把 12 个正典模块从概念节点升级为可教学、可验证、可回写、可交给 Codex / Agent 继续生产的内容资产包。

---

## 1. 适用范围

适用于 12 个正典模块：

```text
A1 发现母体
A2 回到母体
A3 获得原力
A4 显化原力
B1 原力借势
B2 品类独创
B3 模式升维
B4 壁垒锁定
C1 一纸文脉
C2 一个大脑
C3 一张地图
C4 一条链路
```

---

## 2. 标准文件结构

每个模块资产目录必须逐步补齐：

```text
module-canon.md        正典定义
teaching-outline.md    课程大纲
cases.md               案例库
assignments.md         作业与练习
scripts.md             讲稿 / 短视频脚本
evidence.md            真实验证 / 工程证据
evolution.md           迭代记录
```

---

## 3. 每个文件的 Done 标准

| 文件 | Done 标准 |
|---|---|
| module-canon.md | 有定义、不是它、第一性问题、输入、输出、回写、接口 |
| teaching-outline.md | 可直接变成一节课，含目标、结构、练习、验收 |
| cases.md | 至少区分教学样例、真实案例、反例；不得伪造真实验证 |
| assignments.md | 学员能完成，AI/Codex 能辅助批改或整理 |
| scripts.md | 能转成短视频、直播、课程讲稿或社群引导 |
| evidence.md | 明确 evidence strength 与来源，不虚标 |
| evolution.md | 记录本轮改变、剩余缺口、下一次复用方式 |

---

## 4. Evidence 分级

```text
none     尚无证据
seed     工程产物 / 正典依据 / 教学样例
medium   有一次真实使用、学员反馈或可追溯任务链
strong   多次真实验证，能反复复用并产生回写
```

规则：

```text
教学样例 ≠ 真实客户案例。
工程证据 ≠ 市场验证。
Evolution Note ≠ Canon。
Canonical Candidate ≠ Canon。
```

---

## 6. 三个优先样板

本轮先跑通三个样板：

| 模块 | 原因 | 目标 |
|---|---|---|
| A1 发现母体 | 整个体系源头 | 企业家能识别不可复制母体候选 |
| B1 原力借势 | 进入市场入口 | 判断时代为何现在需要我 |
| C3 一张地图 | OS 战略枢纽 | 判断优先级、能力缺口与行动路径 |

---

## 7. Codex 执行边界

Codex 可以执行：

```text
生成骨架
整理案例
生成初稿
补齐模板
同步索引
检查字段完整性
```

Codex 不可自动执行：

```text
批准正典
伪造真实 evidence
修改 CONSTITUTION 母公理
新增第五步
把候选学习写入正式规则
```
