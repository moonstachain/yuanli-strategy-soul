## 2. Sample 001 · CEO 战略仪表盘

### 2.1 Input

```yaml
founder_or_expert_context: 数据分析 / 经营分析服务方
target_user: CEO / 创始人 / 经营负责人
expensive_problem: 经营数据复杂，老板看不懂，看不出战略信号
old_category_or_comparison: BI 报表 / 数据分析 / 可视化大屏
unique_mechanism: 把复杂经营数据转译成 CEO 秒懂的战略信号和下一步动作
evidence_or_signal: B2 external evidence coding includes “数据分析 → CEO 秒懂战略信号” pattern
desired_budget_or_attention_shift: 从买报表转向买战略判断
```

### 2.2 Before

```text
我们帮企业做数据分析和经营报表，让老板更好地看到公司的情况。
```

### 2.3 After

```text
我们帮助 CEO 把复杂经营数据变成一眼能看懂的战略信号和下一步动作。
它不同于传统 BI 报表，因为它不是展示更多数据，而是把经营噪音压缩成老板能直接决策的战略仪表盘。
```

### 2.4 Rationale

```text
The before version is trapped in the old category “数据分析 / 报表”. The after version escapes into “CEO 战略仪表盘”, naming the expensive problem, buyer, mechanism, and budget shift.
```

### 2.5 Unknown Gap

```text
Need target CEOs to test whether “战略仪表盘” is clearer and more valuable than “数据分析报表”.
```

### 2.6 Next Test

```text
Run clarity / repeatability / budget-pull test with 5 target CEOs or founders.
```

## 3. Sample 002 · 高客单服务 AI 工作台

### 3.1 Input

```yaml
founder_or_expert_context: 高客单咨询 / 服务 / 私域成交团队
target_user: 高客单服务型创业者 / 顾问 / 教练 / 高端服务机构
expensive_problem: 服务过程不可视化、成交依赖个人表达、交付不可管理
old_category_or_comparison: 咨询服务 / 销售话术 / CRM / PPT 方案
unique_mechanism: 把高客单服务变成现场可视化、可成交、可管理的 AI 工作台
evidence_or_signal: B2 coded pattern includes “高客单服务 → AI 工作台”
desired_budget_or_attention_shift: 从买顾问时间转向买可复用服务系统
```

### 3.2 Before

```text
我们帮高客单服务团队用 AI 提升成交和交付效率。
```

### 3.3 After

```text
我们帮助高客单服务团队把依赖个人表达的成交和交付，变成现场可视化、可管理、可复用的 AI 工作台。
它不同于销售话术或 CRM，因为它把客户理解、方案生成、成交推进和交付复盘压进同一个可操作系统里。
```

### 3.4 Rationale

```text
The before version is generic AI efficiency language. The after version creates a new category: AI 工作台, and clarifies why it is not CRM or sales copywriting.
```

### 3.5 Unknown Gap

```text
Need service founders to test whether “AI 工作台” triggers stronger understanding and purchase intent than “AI 提效”.
```

### 3.6 Next Test

```text
Run naming test with high-ticket service founders; ask them to repeat what the product is and why it differs from CRM or sales tools.
```

## 4. Sample 003 · AI 原生市场验证报告

### 4.1 Input

```yaml
founder_or_expert_context: 创业者 / 产品负责人 / 市场研究服务
target_user: 早期创业者 / 新产品负责人 / 专家型创业者
expensive_problem: 有机会假设但不知道市场是否真实、用户是否买单、品类是否成立
old_category_or_comparison: 市场调研 / 商业计划书 / 竞品分析
unique_mechanism: 把一句话机会假设转成可决策的 AI 原生市场验证报告
evidence_or_signal: B2 coded pattern includes “一句话机会假设 → AI 市场验证报告”
desired_budget_or_attention_shift: 从买调研资料转向买决策证据
```

### 4.2 Before

```text
我们用 AI 帮创业者做市场调研和竞品分析。
```

### 4.3 After

```text
我们帮助早期创业者把一句话机会假设，变成能判断用户、品类、竞品和购买信号的 AI 原生市场验证报告。
它不同于传统市场调研，因为它不是堆资料，而是把不确定的创业想法压缩成可继续、可放弃或可转向的决策证据。
```

### 4.4 Rationale

```text
The before version is trapped in “AI 市场调研”. The after version creates a decision-oriented category: AI 原生市场验证报告.
```

### 4.5 Unknown Gap

```text
Need founders to test whether “市场验证报告” creates stronger willingness to pay than “市场调研”.
```

### 4.6 Next Test

```text
Run 5 founder interviews comparing the two phrasings and score clarity, relevance, novelty, and budget-pull.
```
