# C2 · 一个大脑 · Teaching Outline 2.0

PMO Issue: #95
Asset Type: teaching-outline
Priority: P0.5

## 一句话目标

帮助学员把知识、案例、经验、任务结果、证据、复盘和正典候选，沉淀成可检索、可综合、可引用、可指出缺口、可生成下一步行动、可持续回写的战略记忆系统。

## 课程定位升级

C2 不只是知识管理课，而是原力 OS 的记忆中枢。

```text
C2 一个大脑 = 可调用的长期记忆 × 有证据的综合判断 × 能回写的战略学习
```

## 适用对象

- 内容资产很多但难以复用的人
- 频繁重复解释和重复生产的人
- 想让 AI 长期理解自己和事业系统的人
- 希望把经验变成团队资产的创始人
- 正在搭建原力 OS、Agent 或 PMO 系统的人

## 学完以后，学员应能完成

1. 区分资料库、知识库、搜索系统和真正的大脑。
2. 设计自己的原力大脑 schema。
3. 建立 Capture、Schema、Retrieval、Synthesis、Graph、Gap、Writeback 七层结构。
4. 用标准 Query Protocol 让大脑输出结论、证据、缺口和下一步行动。
5. 用 Writeback Protocol 把行动结果写回 evidence、evolution、canonical candidate 或 PMO。
6. 写出 C2 原力大脑设计图。

## 课程结构

### 1. 为什么知识库不是大脑

知识库只是存放信息。真正的大脑必须能调用、综合、引用、发现缺口、生成行动，并持续回写。

### 2. C2 在原力 OS 中的位置

```text
C1 一纸文脉：我是谁，我为什么做，边界是什么
C2 一个大脑：我记得什么，什么被验证，哪里还不知道
C3 一张地图：我该往哪里走，哪个缺口优先
C4 一条链路：我如何执行、验证、复盘、回写
```

### 3. 七层原力大脑

```text
Capture 捕获层
Schema 结构层
Retrieval 检索层
Synthesis 综合层
Graph 图谱层
Gap 缺口层
Writeback 回写层
```

### 4. 五个逻辑脑

```text
Founder Brain
Trilogy Brain
Client / Case Brain
PMO Brain
Agent Skill Brain
```

### 5. Query Protocol

每次调用 C2，都要尽量输出：

```text
直接结论
证据来源
关联模块
当前成熟度
未知缺口
下一步 C4 行动
回写建议
```

### 6. Writeback Protocol

大脑回答后，要判断是否写回：

```text
inbox
evidence
evolution note
canonical candidate
maturity scorecard
C4 task
```

### 7. 从 C2 到 C3/C4

C2 发现缺口，C3 排序缺口，C4 执行任务，Evidence 验证结果，C2 再把学习写回。

## 输出作业

```text
C2 原力大脑设计图
```

字段：

- 我的一个大脑服务谁
- 我的输入源有哪些
- 我的 schema 类型有哪些
- 我的检索场景
- 我的综合回答格式
- 我的图谱关系
- 我的缺口类型
- 我的回写规则
- 我的第一个真实查询样例

## 验收标准

- 不是文件夹堆积
- 有清楚 schema
- 有调用场景
- 有证据规则
- 有缺口识别
- 有回写规则
- 能支撑 C3 一张地图与 C4 一条链路
