# C2 · 一个大脑 · Teaching Outline

PMO Issue: #95
Asset Type: teaching-outline

## 一句话目标

帮助学员把知识、案例、经验、任务结果和复盘沉淀成可检索、可调用、可更新的个人或组织大脑。

## 适用对象

- 内容资产很多但难以复用的人
- 频繁重复解释和重复生产的人
- 想让 AI 长期记住自己知识的人
- 希望把经验变成团队资产的创始人

## 学完以后，学员应能完成

1. 区分资料库、知识库和大脑。
2. 设计一个最小大脑结构。
3. 建立输入、标签、检索和回写规则。
4. 把一个项目结果写入大脑。
5. 写出 C2 一个大脑卡。

## 课程结构

### 1. 为什么知识库不是大脑

知识库只是存放信息，大脑必须能调用、判断、更新和复用。

### 2. 一个大脑的四个层级

```text
素材层 / 知识层 / 案例层 / 决策层
```

### 3. 输入规则

第一性问题：什么信息值得进入长期记忆？

### 4. 检索规则

第一性问题：未来我会在什么场景下需要它？

### 5. 回写规则

每次行动之后，都要判断哪些内容要更新到大脑。

## 输出作业

```text
C2 一个大脑卡
```

字段：

- 我的资产类型
- 大脑层级
- 输入标准
- 标签规则
- 检索场景
- 回写规则
- 第一个写入样本

## 验收标准

- 不是文件夹堆积
- 有清楚分类
- 有调用场景
- 有回写规则
- 能进入 C3“一张地图”
