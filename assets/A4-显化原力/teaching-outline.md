# A4 · 显化原力 · Teaching Outline

PMO Issue: #89
Asset Type: teaching-outline

## 一句话目标

帮助学员把内在原力转化为市场可感知、用户可验证、系统可回写的外部成果。

## 适用对象

- 已经提取出核心能力的人
- 想把能力变成产品、内容、服务的人
- 想验证市场反馈的专家型个体
- 准备从原力资产进入原力创业的人

## 学完以后，学员应能完成

1. 选择一个最小显化场景。
2. 把能力转化为可被用户理解的表达。
3. 设计最小产品或最小服务。
4. 收集用户反馈。
5. 写出 A4 显化实验卡。

## 课程结构

### 1. 为什么原力必须显化

没有显化，原力只停留在自我认知；只有进入真实反馈，才会成为资产。

### 2. 显化的四个出口

```text
内容 / 产品 / 服务 / 系统
```

### 3. 从能力到用户问题

第一性问题：我的能力解决了谁的什么具体问题？

### 4. 最小显化实验

```text
一个明确用户 + 一个具体问题 + 一个小交付 + 一次反馈
```

### 5. 回写机制

第一性问题：这次显化之后，我学到了什么？它如何回写到母体和 OS？

## 输出作业

```text
A4 显化实验卡
```

字段：

- 我的核心能力
- 目标用户
- 具体问题
- 最小交付
- 反馈证据
- 回写结论
- 下一步进入 B1 的机会

## 验收标准

- 有真实用户或真实场景
- 有可感知交付
- 有反馈证据
- 有回写结论
- 能进入 B1“原力借势”
