## Naming Criteria

```text
简单
易传播
有对比锚
能让用户一秒理解它是新东西
能承接天然预算
```

## Candidate 1 · 原力品类卡

### Name

```text
原力品类卡
```

### One-line Definition

```text
帮创业者把自己的独特优势，变成用户能理解、能复述、愿意付费的新品类表达。
```

### Target User

```text
专家型个体 / 早期创业者 / 高认知内容创业者
```

### New Problem

```text
我知道自己不一样，但用户总把我归到旧品类里，导致我被比较、被压价。
```

### Old Category It Breaks From

```text
定位咨询 / 品牌命名 / 商业模式咨询
```

### Budget Hypothesis

```text
品牌定位预算 / 战略咨询预算 / 创业陪跑预算
```

## Candidate 2 · 独创品类工作坊

### Name

```text
独创品类工作坊
```

### One-line Definition

```text
一场帮你从旧赛道竞争，转向新品类开创的实战工作坊。
```

### Target User

### New Problem

```text

```

### Old Category It Breaks From

```text
定位课 / 品牌课 / 创业课
```

### Budget Hypothesis

```text
创业学习预算 / 咨询预算 / 个人品牌预算
```

### Name

无法分类实验室
```

### One-line Definition

### Budget Hypothesis

高管教练预算 / 商业咨询预算 / 创新项目预算
```

## Test Questions

对每个候选名称问：

```text
你觉得它解决什么问题？
你觉得它是给谁的？
你会把它归到哪个旧品类？
它和旧品类有什么不同？
你能不能用自己的话复述一句？
你什么情况下愿意继续了解或付费？
```

## Decision Rule

候选名不是看创始人喜欢，而是看：

```text
用户是否理解
用户是否复述
用户是否不再直接归入旧品类
用户是否出现继续了解或付费信号
```
