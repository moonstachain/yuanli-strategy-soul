# B2 · 品类独创 · Workshop Runbook

PMO Issue: #130
Asset Type: workshop-runbook
Date: 2026-06-30

## Purpose

把 B2 品类独创从 validation-pack-ready 推进到外部真实验证。这个 runbook 用于执行 5-10 人小范围验证工作坊。

## Workshop Goal

验证目标用户是否能理解、复述并对 B2 新品类表达产生进一步了解或付费兴趣。

## Target Participants

```text
5-10 个目标用户
专家型个体
早期创业者
传统行业转型者
高认知内容创业者
有明确付费能力的人
```

## Duration

```text
90-120 分钟
```

## Required Inputs

- naming-test.md
- user-language.md
- validation-records.md
- 3 个候选名称
- B2 品类独创卡模板

## Workshop Agenda

### 0-10 min · Context

说明本次不是正式销售，而是验证：

```text
我们在测试一个新品类表达，重点看你是否理解、是否能复述、是否觉得它和你有关。
```

### 10-25 min · Problem Language Capture

让参与者回答：

```text
你现在最想解决的业务/成长/定位问题是什么？
你会如何用自己的话描述它？
你过去用什么旧方法解决？为什么不够？
```

### 25-45 min · Category Naming Test

展示 3 个候选名称，每个名称问：

```text
你觉得它解决什么问题？
它是给谁的？
它和旧品类有什么不同？
你能不能用自己的话复述一句？
```

### 45-70 min · B2 Category Card

每位参与者填写：

```text
新用户
贵问题
旧品类
新品类命名
一句话定义
对立面
为什么现在重要
```

### 70-90 min · Repeatability Test

让参与者互相复述对方的一句话定义。记录：

```text
是否能复述
哪里误解
哪些词最容易被保留
哪些词导致混淆
```

### 90-110 min · Purchase Signal

追问：

```text
什么情况下你愿意继续深入？
什么情况下你愿意付费？
你会拿它和什么旧方案比较？
你觉得最有价值的部分是什么？
```

### 110-120 min · Close

告知：后续会整理用户语言、命名结果和产品化判断。

## Data to Capture

```text
participant type
problem words
old category words
candidate name reactions
repeatable sentences
confusion points
purchase signals
```

## Pass Criteria

```text
至少 5 个命名样本
至少 5 条用户原话
至少 3 个可复述的一句话品类定义
至少 1 个明确付费或进一步了解信号
```

## Fail Criteria

```text
多数用户无法说明这是给谁的
多数用户归入旧品类且无法区别
无法复述一句话定义
没有任何付费或继续了解信号
```

## Decision After Workshop

| Result | Decision |
|---|---|
| Strong repeatability + purchase signal | B2 Validation 1/5 -> 2/5 |
| Repeatable but weak purchase signal | iterate offer/pricing |
| Clear problem but weak name | iterate naming |
| Weak problem relevance | revisit target user / expensive problem |
