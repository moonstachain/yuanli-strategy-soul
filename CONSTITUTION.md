## §0 · 母公理

整套原力战略三部曲只有一条不可再分的根：

> **ʸx：做会自我繁殖的事，不做一次性的事。**

判断一个任务是否属于本体系，不看它是否复杂，而看它是否满足：

```text
完成一次之后，是否让下一次更容易？
完成一个之后，是否能孵化下一个？
完成一轮之后，是否能回写成资产、能力、系统或判断？
```

## §1 · 三部曲裁决

### 1.1 三部曲定义

```text
原力战略三部曲 = 原力资产 / 原力创业 / 原力OS
```

### 1.2 原力战略的位置

```text
原力战略·白皮书 = 公理层 umbrella
```

它坐在三本书之上，回答“为什么整套体系必然成立”，不再作为三本之一。

### 1.3 三部总飞轮

```text
原力资产 → 原力创业 → 原力OS → 原力资产升级
```

第一部生成会复利的原力母体；第二部让母体借势进入市场；第三部把市场与任务经验系统化并回写母体。

## §2 · 三本书边界

| 书 | 方向 | 对象 | 主公式 | 核心结构 | 终态 |
|---|---|---|---|---|---|
| 原力资产 | 向内 | 人 | 原力母体 × AI外化 × 市场验证 × OS回写 | 清算 → 返源 → 成形 → 外化 → 验证 → 回写 → 复利 | 会复利的原力母体 |
| 原力创业 | 向外 | 事 | 财富 = 借势 × 合力 | 原力借势 → 品类独创 → 模式升维 → 壁垒锁定 | 会复利的生意 |
| 原力OS | 向上 | 法 | 文脉 × 大脑 × 地图 × 链路 × 递归回写 | 工作生命体 | ʸx 孵化者 |

三本书的 MECE 原则：

```text
人 ≠ 事 ≠ 法
向内 ≠ 向外 ≠ 向上
找到母体 ≠ 变现母体 ≠ 系统回写母体
```

## §3 · 三大公式

### 3.1 原力资产公式

```text
原力资产 = 原力母体 × AI外化 × 市场验证 × OS回写
```

其中：

```text
原力母体 = 注意力雷达 × 品味函数 × 身体化能力 × 生命经验 × 长期承诺
```

原力母体，是一个人持续生成判断、作品、关系和意义的底层生命结构。

更硬的内部定义：

```text
原力母体 = 一个人的生成函数
```

对外传播用“原力母体”，内部系统化用“生成函数”。

### 3.2 原力创业公式

```text
财富 = 借势 × 合力
```

借势是外部不变量，合力是内部不变量。二者是乘法，任一为 0，整体归零。

创业侧的顺序裁决：

```text
原力资产先完成原力母体识别、AI外化与初步市场验证。
原力创业第一关完成借势，让已验证母体进入时代窗口。
```

### 3.3 原力OS公式

```text
原力OS = 文脉 × 大脑 × 地图 × 链路 × 递归回写
```

白话：

```text
懂你 + 记得住 + 会判断 + 能动手 + 每次变强
```

## §4 · 原力创业第一关裁决

正式裁决：

```text
“原力觉醒”归入《原力资产》。
“原力借势”成为《原力创业》第一关。
```

新四关：

```text
原力借势 → 品类独创 → 模式升维 → 壁垒锁定
```

四关获得物：

| 关 | 第一性问题 | 获得物 |
|---|---|---|
| 原力借势 | 这个时代为什么现在需要我？ | 势能地图 |
| 品类独创 | 市场如何认得我？ | 心理账户 / 品类入口 |
| 模式升维 | 怎么持续赚钱、可复制？ | 利润容器 |
| 壁垒锁定 | 怎么守住、不被拿走？ | 财富飞轮 |

## §5 · 原力OS 最新裁决

原力OS 不再以“四部”为第一主轴，而以“四组件”为第一主轴。

### 5.1 OS 本体层

| 组件 | 白话 | 本质 |
|---|---|---|
| 文脉 | 灵魂 | 我是谁 |
| 大脑 | 记忆 | 我知道什么 |
| 地图 | 判断 | 我怎么决策 |
| 链路 | 手脚 | 我怎么行动 |

### 5.2 OS 宪法层

三法印：

```text
递归进化 / 技能统帅 / 人类友好
```

三法印不是普通章节，而是 OS 的底层约束。

### 5.3 OS 成熟层

S 路径：

```text
S1 建模：1·x，学生
S2 工具：x·y，工匠
S3 板：x^y，产品人
S4 舱：ʸx，孵化者
```

### 5.4 OS 教学层

四部保留为教学路径：

```text
信念：为什么转
图景：转向哪
工具：怎么转
身份：转完是谁
```

## §6 · 共享脊骨

跨三部共有六条脊骨：

| 脊骨 | 作用 |
|---|---|
| ʸx | 母公理 |
| 原力母体 | 资产主根 |
| AI外化 / 市场验证 / OS回写 | 资产成为复利系统的机制链 |
| 借势 × 合力 | 财富形态 |
| 四关 | 市场动作链 |
| 文脉 / 大脑 / 地图 / 链路 / 递归回写 | OS 生命循环 |

任何新概念若不能挂回至少一条脊骨，默认不得进入正典层。

## §7 · 三部接口

```text
原力资产 → 原力创业：
原力母体定义、注意力雷达、品味函数、身体化能力、生命经验、长期承诺、天选用户线索、贵问题线索、市场验证记录。

原力创业 → 原力OS：
势能地图、心理账户、品类表达、利润容器、交付链路、数据反馈、壁垒控制点。

原力OS → 原力资产：
文脉回写、大脑回写、地图回写、链路回写、母体升级。
```

## §8 · 衍生任务裁决链

新任务进入本仓，按以下顺序裁决：

```text
母公理 → 三部边界 → 共享脊骨 → 任务归属 → 输出物 → 递归回写
```

禁止反向操作：不能因为一个任务有趣，就倒逼修改母公理。
