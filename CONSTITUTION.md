## §0 · 母公理

整套原力战略三部曲只有一条不可再分的根：

> **ʸx：做会自我繁殖的事，不做一次性的事。**

v0.9 对母公理的解释：

```text
任何概念，若不能被执行、验证、回写、复用，就不得进入正典。
任何输出，若不能成为下一次的输入，就不是原力资产。
任何系统，若不能让下一轮更强，就不是原力OS。
任何地图，若不能指导取舍、配置能力、识别瓶颈和生成行动链路，就不是原力OS的一张地图。
```

## §2 · 三部总定义

长定义：

```text
原力战略三部曲，是一套以 ʸx 为母公理，把一个人的原力母体发现、回归、激活、显化，并进一步借势创业、锁定壁垒，最终写入文脉、大脑、战略地图与行动链路，形成可递归进化的个人与事业复利系统。
```

短定义：

```text
原力战略三部曲 = 找到母体，借势创业，系统回写。
```

最短定义：

```text
母体 → 原力 → 创业 → OS → 母体升级
```

## §3 · 三本书边界

| 书 | 方向 | 对象 | 四步 | 终态 |
|---|---|---|---|---|
| 原力资产 | 向内 | 人 | 发现母体 → 回到母体 → 获得原力 → 显化原力 | 可进入市场与OS的原力 |
| 原力创业 | 向外 | 事 | 原力借势 → 品类独创 → 模式升维 → 壁垒锁定 | 会复利、守得住的生意 |
| 原力OS | 向上 | 法 | 一纸文脉 → 一个大脑 → 一张地图 → 一条链路 | 会递归进化的工作生命体 |

边界裁决：

```text
原力资产的“显化原力” = 市场验证 / 创业接口。
原力创业负责完整商业化与变现系统。
原力OS的“递归回写”是贯穿四步的底层机制，不作为第五步破坏结构。
原力OS的“一张地图” = 战略地图 / 能力组件地图 / 事业系统地图，不是普通思维导图。
```

## §4 · 三部核心公式

### 4.1 原力资产

```text
原力资产 = 原力母体 × 原力激活 × 真实显化 × OS回写
```

其中：

```text
原力母体 = 注意力雷达 × 品味函数 × 身体化能力 × 生命经验 × 长期承诺
```

### 4.2 原力创业

```text
财富 = 借势 × 合力
```

其中：

```text
合力 = 已被发现、回归、激活、显化的原力。
借势 = 时代、结构、入口与母体的同向共振。
```

### 4.3 原力OS

```text
原力OS = 一纸文脉 × 一个大脑 × 一张地图 × 一条链路 × 递归回写
```

白话：

```text
懂你 + 记得住 + 会判断 + 能行动 + 每次变强
```

四模块正典定义：

| 模块 | 本质 | 解决的问题 | 终态产物 |
|---|---|---|---|
| 一纸文脉 | 灵魂 / 约束 | AI 如何真正懂我？ | 最小正典文件 |
| 一个大脑 | 记忆 / 索引 | 系统应该记住什么？ | 可检索记忆系统 |
| 一张地图 | 战略判断 / 能力组件 | 系统如何判断方向、取舍、优先级和能力缺口？ | 战略地图 / 能力组件地图 |
| 一条链路 | 行动流 / 执行系统 | 系统如何把判断变成可执行流程？ | OSA / SOP / Agent / Codex 链路 |

特别裁决：

```text
一张地图不是思维导图，不是信息可视化，不是漂亮但不能决策的图。
一张地图是战略地图 / 能力组件地图 / 事业系统地图。
它必须能识别战略控制点、支撑能力、瓶颈能力、可外包能力、必须自持能力与下一步行动路径。
```

## §6 · 三部接口

```text
原力资产 → 原力创业：
母体候选、五部件卡组、原力能力包、天选用户线索、贵问题线索、市场验证记录、创业输入包。

原力创业 → 原力OS：
势能地图、新品类定义、利润容器、财富飞轮、客户成功案例、数据资产、壁垒控制点。

原力OS → 原力资产：
文脉回写、大脑回写、地图回写、链路回写、母体升级记录。
```

## §7 · 递归闭环

```text
资产闭环：发现母体 → 回到母体 → 获得原力 → 显化原力 → 母体升级
创业闭环：原力输入包 → 借势 → 独创 → 升维 → 锁定 → OS回写包
OS闭环：一纸文脉 → 一个大脑 → 一张战略地图 → 一条链路 → 任务执行 → 递归回写
总飞轮：母体 → 原力 → 市场 → 生意 → OS → 母体升级
```
