# C2 · 一个大脑 · Strategy v1

Date: 2026-06-30
Module: C2 一个大脑
PMO Issue: #95
Priority: P0.5

## Strategic Upgrade

C2 不再只是原力 OS 中的一节“知识管理课”，而是整个原力战略三部曲的记忆中枢。

```text
C2 一个大脑 = 可调用的长期记忆 × 有证据的综合判断 × 能回写的战略学习
```

## Why P0.5

C2 从 P1 提升为 P0.5，理由是：

1. 没有 C2，大量内容资产无法持续复利。
2. 没有 C2，C3 地图和 C4 链路容易变成机械看板。
3. 没有 C2，PMO 复盘没有长期记忆。
4. 没有 C2，原力 OS 无法知道什么被验证、什么仍是未知。
5. 没有 C2，AI 只是工具，不是共同进化系统。

## Position in Yuanli OS

```text
C1 一纸文脉：定义我是谁、为什么做、边界是什么
C2 一个大脑：记住我做过什么、什么被验证、哪里还不知道
C3 一张地图：判断下一步往哪里走、哪个缺口优先
C4 一条链路：执行、验证、复盘、回写
```

## One-sentence Definition

C2 一个大脑，是原力 OS 的长期记忆与证据综合层。它把文脉、内容、案例、项目、客户、决策、证据、复盘和正典候选统一沉淀为可检索、可综合、可引用、可指出缺口、可生成下一步行动的战略记忆系统。

## 7-layer Architecture

### 1. Capture Layer

统一捕获外部材料、课程资产、客户案例、AI 对话、项目复盘和 PMO 输出。

### 2. Schema Layer

把输入材料归类为 canon、module、case、assignment、script、evidence、evolution note、decision、project、client、source 等类型。

### 3. Retrieval Layer

支持 keyword、vector、frontmatter、graph edge、time、source strength、canon status、evidence strength 等混合检索。

### 4. Synthesis Layer

不是返回文件，而是输出有引用、有判断、有缺口、有下一步行动建议的答案。

### 5. Graph Layer

建立模块、资产、案例、证据、演化、任务、正典候选之间的关系。

### 6. Gap Layer

自动识别缺失证据、缺真实案例、缺作业、缺验证、claim 过期、边界冲突和商业化缺口。

### 7. Writeback Layer

把行动结果写回 inbox、evidence、evolution note、canonical candidate、maturity scorecard 或 C4 task。

## Core Output Pattern

每次 C2 被调用，标准输出应包含：

```text
1. 直接结论
2. 证据来源
3. 关联模块
4. 当前成熟度
5. 未知缺口
6. 下一步 C4 行动
7. 回写建议
```

## Five Logical Brains

C2 底层是一个大脑，但产品上应分为五个逻辑脑：

1. Founder Brain：明哥创始人大脑
2. Trilogy Brain：三部曲内容大脑
3. Client / Case Brain：客户与案例大脑
4. PMO Brain：项目治理大脑
5. Agent Skill Brain：Agent 技能大脑

## Strategic Goal

```text
永不丢失你的原力文脉。
```

C2 的最终目标不是“帮我存东西”，而是让整个原力战略三部曲变成 queryable、auditable、actionable、self-improving 的战略系统。
