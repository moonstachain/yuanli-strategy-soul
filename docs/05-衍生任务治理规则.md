# 衍生任务治理规则

> 本文件用于管理所有从原力战略三部曲衍生出来的内容、产品、代码、Agent、课程、网站、看板、自动化任务。

---

## 1. 任务分层

| 层级 | 类型 | 例子 |
|---|---|---|
| L0 公理层 | 概念宪法 | ʸx、三部边界、核心公式 |
| L1 概念层 | 概念体系 | 原力U、四关、OS四组件 |
| L2 内容层 | 写作表达 | 章节、公众号、课程稿 |
| L3 产品层 | 可交互系统 | 网站、看板、Agent、知识库 |
| L4 执行层 | 具体任务 | Codex 实现、Bug 修复、页面生成 |
| L5 回写层 | 复盘沉淀 | 模板、SOP、skill、决策记录 |

---

## 2. 任务启动五问

任何任务先回答：

```text
1. 它属于原力资产、原力创业、原力OS，还是跨三部？
2. 它服务哪条母公理或共享脊骨？
3. 它的输入是什么？
4. 它的输出物是什么？
5. 它如何递归回写，成为下一次任务的资产？
```

## 4. 优先级规则

| 优先级 | 定义 |
|---|---|
| P0 | 影响母公理、三部边界、核心公式 |
| P1 | 影响多个衍生任务的共用结构 |
| P2 | 影响单个产品 / 内容 / 页面 |
| P3 | 优化表达、格式、细节 |

---

## 5. 验收规则

一个任务完成，不以“做完”为准，而以“可复用”为准。

必须至少交付以下之一：

```text
正典文档
模板
SOP
Prompt
代码模块
数据结构
看板规则
复盘记录
```

---

## 6. 递归回写规则

任务结束必须回写：

| 输出类型 | 回写位置 |
|---|---|
| 新概念 | docs/ 或 CONSTITUTION.md |
| 新判断 | docs/04-跨三部脊骨矩阵.md 或对应分册 |
| 新流程 | codex/ 或 project/ |
| 新产品方向 | project/KANBAN_TEMPLATE.md 或 Issue |
| 新自动化 | codex/PROMPT_PACK.md 或独立仓 |

---

## 7. 禁止事项

```text
禁止无归属任务
禁止无输出物任务
禁止无回写任务
禁止把一次性劳动包装成系统工程
禁止为了新想法反向污染母公理
禁止让 Codex 在没有验收标准的情况下执行
```
