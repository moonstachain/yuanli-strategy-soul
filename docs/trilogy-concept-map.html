三部曲递进逻辑

不是三个主题并列，而是一条不可替换的生命链。

Part I

原力资产

发现母体 → 回到母体 → 获得原力 → 显化原力

解决：我从哪里来。终态：可进入市场与OS的原力。

Part II

原力创业

原力借势 → 品类独创 → 模式升维 → 壁垒锁定

解决：我如何进入世界。终态：会复利、守得住的生意。

Part III

原力OS

一纸文脉 → 一个大脑 → 一张地图 → 一条链路

解决：如何让每一次进入世界，都让下一次更强。

特别裁决：一张地图不是思维导图

原力OS里的“一张地图”，是战略地图 / 能力组件地图 / 事业系统地图。它接近 IBM CBM 的精神，但用于个人与事业OS：把事业系统拆成可判断、可取舍、可配置、可执行、可升级的能力组件。

三部接口与递归闭环

资产 → 创业

母体候选、原力能力包、天选用户、贵问题、市场验证记录、创业输入包。

创业 → OS

势能地图、新品类定义、利润容器、财富飞轮、客户案例、OS回写包。

OS → 母体升级

文脉回写、大脑回写、地图回写、链路回写，让下一轮更强。

最高验收

一个任务完成后，必须问： 它是否更新了一纸文脉？ 它是否沉淀进一个大脑？ 它是否更新了一张战略地图？ 它是否强化了一条链路？ 它是否让下一次更强？ 若五个答案都是否，它只是一次性劳动，不属于原力OS。
