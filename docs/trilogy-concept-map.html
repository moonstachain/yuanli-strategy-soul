第一部：原力资产

从同质化能力中拆出不可复制的源头，让“我”成为可以进入市场与系统的原力。

A1Source

发现母体

识别持续生成判断、作品、关系和意义的底层生命结构。

不是兴趣、人设、职业标签、人格测试。问题AI 清算之后，我身上还剩什么会持续生成差异？输出母体候选、母题清单、母体一句话。回写文脉 / 母体卡。
A2Return

回到母体

停止被外部评价、短期机会和工具兴奋牵引，以母体重组选择与边界。

不是退回舒适区、自我感动、排斥市场。问题如果以母体为中心，哪些事保留，哪些事停止？输出五部件卡组、不再做清单、边界。回写文脉 / 战略边界。
A3Activate

获得原力

让母体被训练、AI 外化、作品化、关系化，形成可调用的高能能力。

不是突然开悟、神秘能量、学更多工具。问题母体如何变成判断力、表达力、创造力、连接力？输出原力能力包、方法论模型、作品样本。回写大脑 / 样例库。
A4Manifest

显化原力

把原力放进真实世界，验证是否被人需要、相信、复用、付费和共建。

不是完整商业化，不是创业全部。问题这个原力是否真的被世界需要？输出市场验证记录、创业输入包、母体回写。回写机会库 / 原力借势。

第二部：原力创业

把已经显化的原力，放进时代势能、用户心智、商业模式与系统壁垒。

B1Leverage

原力借势

判断这个时代为什么现在需要我的原力，并找到最适合进入市场的势能窗口。

不是追风口、蹭热点、宏观叙事自嗨。问题这个时代为什么现在需要我？输出势能地图、机会假设、入场窗口。回写机会地图 / 长期承诺。
B2Category

品类独创

把原力翻译成市场能识别、能记住、愿意付费、愿意转述的新入口。

不是起名字、包装卖点、差异化话术。问题市场如何认得我？输出新品类定义、价值主张、心理账户。回写文脉词典 / 用户语言库。
B3Model

模式升维

把被市场认出的原力品类，升级为可复制、可复利、可扩张的利润容器。

不是卖更多时间、简单涨价、堆产品。问题如何持续赚钱、可复制？输出利润容器、产品阶梯、三链路画布。回写OS 链路 / 数据库。
B4Moat

壁垒锁定

让跑通的利润容器不被轻易复制、拿走、替代，并能越跑越强。

不是自我安慰式护城河、保密、先发幻觉。问题如何守住、不被拿走？输出财富飞轮、壁垒地图、OS 回写包。回写地图 / 壁垒库 / 母体升级。

第三部：原力OS

把成事经验写成系统，让每一次进入世界，都让下一次更强。

C1Context

一纸文脉

把我是谁、为什么做、原力母体、长期承诺、品味函数、战略边界和表达风格，写成最小正典文件。

不是个人简介、品牌文案、愿景口号、提示词集合。问题AI 如何真正懂我，并保持我的判断边界？输出一纸文脉、说明书 V3、AI 使用协议。回写CONSTITUTION / 元AI上下文。
C2Brain

一个大脑

把知识、案例、术语、样例、数据、经验和任务结果，沉淀成可检索、可调用、可版本化的记忆系统。

不是文件夹堆积、资料收藏、知识库幻觉。问题系统应该记住什么，才能让下一次不从零开始？输出知识索引、案例索引、术语表、样例库。回写source/indexes / extracts。
C3Map

一张地图

把文脉、大脑、市场验证和创业结果，组织成可判断方向、取舍、优先级、能力缺口和行动路径的战略地图。

不是思维导图、信息可视化、漂亮但不能决策的图。问题系统如何判断方向、取舍、优先级和能力缺口？输出战略地图、能力组件地图、事业系统地图。回写OS 地图 / 项目看板 / Codex 树。
C4Chain

一条链路

把战略地图中的判断，转化为可重复执行、可交给人或 AI、可验收、可回写、可自动化的行动流程。

不是待办事项、临时任务、一次性交付。问题如何把战略判断变成可复用、可执行、可自动化的行动？输出SOP、Agent workflow、Codex-ready task。回写codex / project / templates / skills。

特别校准：一张地图不是普通地图

这是 v0.9 的关键升级。原力OS里的地图，本质是战略地图 / 能力组件地图 / 事业系统地图，接近 IBM CBM 的精神，但服务于个人与事业的原力系统。

普通地图

回答：这里有什么？

普通思维导图、信息图、结构图，通常用于展示信息。它可以清楚，但不一定能帮你取舍。

原力OS地图

回答：什么最重要？

战略地图必须识别战略控制点、支撑能力、瓶颈能力、可外包能力、必须自持能力，并生成下一步行动链路。

一张地图 = 目标地图 + 用户地图 + 机会地图 + 能力组件地图 + 控制点地图 + 因果地图 + 商业地图 + 决策阈值 完成标志：系统知道先做什么、不做什么、哪里是瓶颈、哪里是控制点。

三部接口地图

三部不是三本孤立的书，而是一套连续的递归工程。

原力资产 → 原力创业

  • 母体候选
  • 五部件卡组
  • 原力能力包
  • 天选用户线索
  • 贵问题线索
  • 市场验证记录
  • 创业输入包

原力创业 → 原力OS

  • 势能地图
  • 新品类定义
  • 利润容器
  • 财富飞轮
  • 客户成功案例
  • 数据资产
  • 壁垒控制点

原力OS → 原力资产

  • 文脉回写
  • 大脑回写
  • 战略地图回写
  • 行动链路回写
  • 母体升级记录
  • 下一轮原力更清楚

递归闭环

资产闭环：发现母体 → 回到母体 → 获得原力 → 显化原力 → 母体升级 创业闭环：原力输入包 → 借势 → 独创 → 升维 → 锁定 → OS 回写包 OS闭环：一纸文脉 → 一个大脑 → 一张战略地图 → 一条链路 → 任务执行 → 递归回写 总飞轮：母体 → 原力 → 市场 → 生意 → OS → 母体升级
