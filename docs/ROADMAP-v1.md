## 1. 母公理

```text
ʸx：做会自我繁殖的事，不做一次性的事。
```

本路线图下，任何任务必须满足：

```text
输入清楚
输出清楚
验收清楚
回写清楚
下一次复用方式清楚
```

否则不得进入 Done。

## 2. v1.0 阶段定义

当前系统已经完成：

```text
正典骨架
公开门户
十二模块数据化
Asset / Startup / OS 工具页雏形
项目看板模板
Codex 任务模板
```

v1.0 的核心不是继续扩概念，而是进入：

```text
正典 → 文脉 → 大脑 → 地图 → 链路 → 执行 → 回写
```

最终形成：

```text
原力战略白板 OS = 正典地图 × 事业能力组件地图 × 任务链路 × 递归回写
```

### Phase 2 · 文脉白板 v1.0

目标：从“正典门户”升级为“战略白板”。

输出：



目标：让系统不失忆。

输出：


规则：


## 5. v1.0 Done 标准

```text
1. 白板可打开、可导航、可表达 12 个正典模块。
2. 每个模块能显示定义、不是它、第一性问题、输入、判断、输出、回写、下一步。
3. 每个白板节点能生成 Codex-ready 任务。
4. 每个任务有验收标准和回写位置。
5. 完成任务必须写 Evolution Note。
6. Canon Map / Whiteboard / Asset / Startup / OS 导航一致。
7. 风格符合 Deep-Space Cockpit：暗色、暖金、高信息密度、稳定面板、少装饰、多判断。
```
