原力OS · 四组件

四组件是本体层；三法印是宪法层；S路径是成熟层；四部是教学层。

Soul

文脉

灵魂：我是谁，为什么做，边界是什么。缺文脉，AI 不懂你。

Memory

大脑

记忆：知识、案例、经验、判断沉淀。缺大脑，每次从零开始。

Judgment

地图

判断：因果图、战略图、关键战役。缺地图，系统不会决策。

Action

链路

手脚：OSA、agent、workflow、自动化执行。缺链路，只想不做。

任务 → 输出 → 复盘 → 模式 → 技能 → 回写四组件 → 下一次任务更强

文脉回写新洞察、新边界、新自我理解。
大脑回写新资料、新案例、新知识结构。
地图回写新因果、新判断、新关键战役。
链路回写新 SOP、新 Prompt、新 Agent。
飞轮回写让下一次更容易、更高级、更可复用。

Codex 执行前五问

这个任务属于哪一部，服务哪条脊骨？
输入材料、相关仓库、不可改内容是否明确？
输出物是文件、页面、脚本、模板还是测试？
验收标准是否可以逐条勾选？
完成后回写到文脉、大脑、地图还是链路？

Done 的定义

输出物已交付。
验收标准已满足。
回写位置已明确或已更新。
下次复用方式已写明。
必要时已分发到正确执行仓。
