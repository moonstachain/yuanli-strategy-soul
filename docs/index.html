三部曲 · 人 / 事 / 法

原力战略白皮书坐在公理层，三部曲分别落在“向内的人、向外的事、向上的法”。

Part I · Asset

原力资产

公式：你 × AI。把“你”这颗最贵算法解码出来，经过原力U，形成会复利的你。

Part II · Startup

原力创业

公式：借势 × 合力。把原力资产送进市场、现金流与壁垒系统，形成会复利的生意。

Part III · OS

原力OS

公式：文脉 × 大脑 × 地图 × 链路 × 回写。把做事系统变成会自我繁殖的工作生命体。

原力OS · 四组件

四组件是本体层；三法印是宪法层；S路径是成熟层；四部是教学层。

Soul

文脉

灵魂：我是谁，为什么做，边界是什么。缺文脉，AI 不懂你。

Memory

大脑

记忆：知识、案例、经验、判断沉淀。缺大脑，每次从零开始。

Judgment

地图

判断：因果图、战略图、关键战役。缺地图，系统不会决策。

Action

链路

手脚：OSA、agent、workflow、自动化执行。缺链路，只想不做。

任务 → 输出 → 复盘 → 模式 → 技能 → 回写四组件 → 下一次任务更强

文脉回写新洞察、新边界、新自我理解。
大脑回写新资料、新案例、新知识结构。
地图回写新因果、新判断、新关键战役。
链路回写新 SOP、新 Prompt、新 Agent。
飞轮回写让下一次更容易、更高级、更可复用。

唯一入口仓制度

以后所有新任务，先开 Issue，再裁决归属，最后分发到具体内容仓、网站仓或 Codex 执行仓。

01 Issue所有新想法先进本仓 Universal Intake。
02 裁决资产 / 创业 / OS / 跨三部 / 治理。
03 拆解输入、输出、验收、禁止事项。
04 分发内容仓、网站仓、Codex 仓、Agent 仓。
05 回写文脉、大脑、地图、链路、模板。

看板结构

Codex 执行前五问

这个任务属于哪一部，服务哪条脊骨？
输入材料、相关仓库、不可改内容是否明确？
输出物是文件、页面、脚本、模板还是测试？
验收标准是否可以逐条勾选？
完成后回写到文脉、大脑、地图还是链路？

Done 的定义

输出物已交付。
验收标准已满足。
回写位置已明确或已更新。
下次复用方式已写明。
必要时已分发到正确执行仓。
