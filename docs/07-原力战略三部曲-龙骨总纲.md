## 1. 最高龙骨：只做会自我繁殖的事

原力战略三部曲的最高龙骨只有一句话：

```text
ʸx：做会自我繁殖的事，不做一次性的事。
```

这不是一句口号，而是整个体系的最高裁决标准。

任何概念，若不能被执行、验证、回写、复用，就不得进入正典。

任何输出，若不能成为下一次输入，就不是原力资产。

任何系统，若不能让下一轮更强，就不是原力 OS。

任何地图，若不能指导取舍、配置能力、识别瓶颈并生成行动链路，就不是原力 OS 的一张地图。

因此，原力战略三部曲的本质不是课程，不是书稿，不是知识库，而是：

```text
把一个人的不可复制性，训练成能复利、能变现、能系统化、能递归进化的事业生命体。
```

## 2. 三节主脊：人、事、法

原力战略三部曲的主结构，是三节脊柱：

```text
原力资产：向内，解决“我是谁”
原力创业：向外，解决“世界为什么现在需要我”
原力 OS：向上，解决“如何让每一次做事都让下一次更强”
```

### 2.1 第一部：原力资产

原力资产，是把「你」从同质化能力、职业标签、外部评价和二手共识中解码出来，找到持续生成判断、作品、关系和意义的原力母体。

它解决的不是：

```text
我会什么？
```

而是：

```text
为什么我会持续生成这些不同于别人的判断、作品、关系和意义？
```

终态：

```text
可进入市场与 OS 的原力。
```

### 2.2 第二部：原力创业

原力创业，是把已经显化的原力放进时代势能、用户心智、商业模式和系统壁垒中，形成会复利、守得住、可放大的生意。

它解决的不是：

```text
我做什么项目？
```

而是：

```text
这个时代为什么现在需要我？
市场如何认得我？
我如何持续赚钱且可复制？
我如何守住、不被拿走？
```

终态：

```text
会复利、守得住的生意。
```

### 2.3 第三部：原力 OS

原力 OS，是把原力母体、创业经验、知识案例、战略地图和行动链路，装配成可持续递归回写、持续进化的工作生命体。

它解决的不是：

```text
我有哪些工具？
```

而是：

```text
如何让每一次任务、交付、判断、内容、客户案例，都成为下一次能力？
```

终态：

```text
会递归进化的工作生命体。
```

## 4. A 轴：原力资产 —— 母体如何变成原力

### A1 · 发现母体

第一性问题：

```text
AI 清算所有同质化能力之后，我身上还剩什么会持续生成差异？
```

核心动作：

```text
从兴趣、人格、职业优势、人设包装、短期灵感中退出来，识别持续生成判断、作品、关系和意义的底层生命结构。
```

输出：

```text
母体候选清单
母题清单
同质化能力清算表
母体一句话假设
下一步返源问题
```

### A2 · 回到母体

第一性问题：

```text
如果以母体为中心，哪些事要保留，哪些事要停止？
```

核心动作：

```text
停止被外部评价、短期机会、工具兴奋和路径惯性牵引，重新以原力母体为中心组织注意力、品味、能力、经验和长期承诺。
```

输出：

```text
原力母体总卡
注意力雷达卡
品味函数卡
身体化能力卡
生命经验炼金卡
长期承诺卡
不再做清单
```

### A3 · 获得原力

第一性问题：

```text
母体如何变成可调用的判断力、表达力、创造力、连接力和行动力？
```

核心动作：

```text
让原力母体经过训练、AI 外化、作品化、关系化和反馈闭环，形成真实任务中可以调用的高能能力包。
```

输出：

```text
原力能力包
母体说明书
核心概念库
方法论模型
作品样本
AI 外化样例库
```

### A4 · 显化原力

第一性问题：

```text
这个原力是否真的被世界需要？
```

核心动作：

```text
把已经获得的原力放进真实世界，验证是否有人靠近、相信、付费、复用、改变行动并愿意共同建设。
```

输出：

```text
天选用户线索
贵问题线索
心理账户线索
市场验证记录
创业输入包
母体回写记录
```

## 5. B 轴：原力创业 —— 原力如何变成生意

原力创业的核心公式：

```text
财富 = 借势 × 合力
```

其中：

```text
合力 = 已被发现、回归、激活、显化的原力。
借势 = 时代、结构、入口与母体的同向共振。
```

### B1 · 原力借势

第一性问题：

```text
这个时代为什么现在需要我？
```

核心动作：

```text
判断宏周期势、中结构势、微入口势与合力匹配，找到最适合进入市场的势能窗口。
```

输出：

```text
势能地图
机会假设
入场窗口判断
机会优先级
原力借势卡
```

### B2 · 品类独创

第一性问题：

```text
市场如何认得我？
```

核心动作：

```text
把被时代势能放大的原力，翻译成目标用户能理解、能记住、愿意付费、愿意转述并能与旧方案区分的新市场入口。
```

输出：

```text
新品类定义
一句话价值主张
心理账户定位
品类命名候选
用户语言库
内容表达框架
```

### B3 · 模式升维

第一性问题：

```text
如何持续赚钱、可复制？
```

核心动作：

```text
把新品类中的价值承诺，装进前链路、后链路、财链路、产品阶梯与人机分工，形成持续赚钱、可复制、可放大的利润容器。
```

输出：

```text
利润容器
产品阶梯
三链路画布
交付 SOP
增长指标
人机分工表
```

### B4 · 壁垒锁定

第一性问题：

```text
如何守住、不被拿走？
```

核心动作：

```text
把已经跑通的利润容器，进一步沉淀为虚壁垒、实壁垒、入壁垒、出壁垒、飞轮壁垒和母体壁垒。
```

输出：

```text
财富飞轮
壁垒地图
16 控制点
资产护城河
OS 回写包
```

## 6. C 轴：原力 OS —— 生意如何变成系统

原力 OS 的核心公式：

```text
原力 OS = 一纸文脉 × 一个大脑 × 一张地图 × 一条链路 × 递归回写
```

白话：

```text
懂你 + 记得住 + 会判断 + 能行动 + 每次变强
```

### C1 · 一纸文脉

第一性问题：

```text
AI 如何真正懂我，并持续保持我的判断边界？
```

核心动作：

```text
把我是谁、为什么做、原力母体、长期承诺、品味函数、战略边界和表达风格，写成 AI、团队与未来自己都能理解和调用的最小正典文件。
```

输出：

```text
一纸文脉
我的说明书 V3
AI 使用协议
战略边界清单
风格词典
禁止事项清单
```

### C2 · 一个大脑

第一性问题：

```text
系统应该记住什么，才能让下一次不从零开始？
```

核心动作：

```text
把知识、案例、术语、样例、数据、经验和任务结果，沉淀成可检索、可调用、可版本化、可更新的记忆系统。
```

输出：

```text
一个大脑
知识索引
案例索引
术语表
样例库
数据记录
版本记录
```

### C3 · 一张地图

第一性问题：

```text
系统如何知道什么更重要、先做什么、如何取舍、能力缺口在哪里？
```

核心动作：

```text
把文脉、大脑、市场验证和创业结果，组织成一张可判断方向、取舍、优先级、能力缺口和行动路径的战略地图 / 能力组件地图 / 事业系统地图。
```

特别裁决：

```text
一张地图不是思维导图。
一张地图不是信息可视化。
一张地图不是漂亮但不能决策的图。
一张地图 = 战略地图 / 能力组件地图 / 事业系统地图。
```

它必须回答：

```text
什么最重要？
什么是战略控制点？
什么是支撑能力？
什么是瓶颈能力？
什么可外包？
什么必须自己掌握？
什么先做？
什么不做？
```

输出：

```text
一张战略地图
能力组件地图
事业系统地图
机会地图
商业地图
变量因果图
行动阈值
战略路径图
```

核心动作：

把战略地图中的判断，转化为可重复执行、可交给人或 AI、可验收、可回写、可自动化的行动流程。

## 7. 六条横向脊骨：保证三部不散

三部曲不是三本散书，而是同一条母公理在「人、事、法」三面墙上的投影。

所有衍生任务必须至少挂回一条共享脊骨。

| 脊骨 | 作用 | 原力资产 | 原力创业 | 原力 OS |
|---|---|---|---|---|
| ʸx | 母公理 | 资产要会自我繁殖 | 生意要形成财富飞轮 | 系统要孵化下一代智能 |
| 你 × AI | 主根种子 | 第一性公式 | 合力来源 | 文脉与元 AI 的源头 |
| 借势 × 合力 | 财富形态 | 合力来自母体与原力 | 创业核心公式 | 提升执行杠杆与系统化能力 |
| 四关 | 市场动作链 | 觉醒阶段提供母体 | 创业主轴 | 作为商业母语复用 |
| 信息五层 / OSA | 判断底盘 | 找到“你是谁”的全局最优 | 找到打法的全局最优 | 形成地图与链路 |
| 递归回写 | 生命循环 | 复盘自己，升级说明书 | 复盘生意，升级模型 | 复盘任务，升级四组件 |

每个任务必须先判断：

```text
它主要服务哪条脊骨？
它主要落在哪一部？
它是否需要跨三部协同？
它的回写位置在哪里？
```

## 8. 真正的递进关系：不是线性三本书，而是生命飞轮

原力战略三部曲不是：

```text
第一本讲完 → 第二本讲完 → 第三本讲完
```

它真正的递进关系是：

```text
母体 → 原力 → 市场 → 生意 → OS → 母体升级
```

三条闭环：

```text
资产闭环：发现母体 → 回到母体 → 获得原力 → 显化原力 → 母体升级
创业闭环：原力输入包 → 借势 → 独创 → 升维 → 锁定 → OS 回写包
OS 闭环：一纸文脉 → 一个大脑 → 一张战略地图 → 一条链路 → 任务执行 → 递归回写
```

总飞轮：

```text
母体 → 原力 → 市场 → 生意 → OS → 母体升级
```

这意味着：

```text
原力资产不是个人成长。
原力创业不是做项目。
原力 OS 不是工具箱。
```

它们共同构成一个递归系统：

```text
第一轮找到你，第二轮验证你，第三轮系统化你，第四轮升级你。
```

## 10. 龙骨级验收标准

任何新增概念、页面、课程、章节、工具、模板、Agent、Codex 任务，必须通过以下验收。

### 10.1 究竟

是否能回到唯一母公理？

```text
ʸx：做会自我繁殖的事，不做一次性的事。
```

### 10.2 完备

是否覆盖或明确归属到：

```text
人 / 事 / 法
原力资产 / 原力创业 / 原力 OS
```

### 10.3 递进

是否服务于：

```text
找到母体 → 获得原力 → 借势创业 → 锁定壁垒 → 系统回写
```

### 10.4 原子

是否回答每个模块必须回答的问题：

```text
它是什么？
它不是什么？
它解决什么第一性问题？
输入是什么？
判断标准是什么？
输出是什么？
回写到哪里？
下一步接口是什么？
如何让下一次更强？
```

### 10.5 递归

是否明确：

```text
输出物是什么？
回写位置在哪里？
下次如何复用？
```

如果不能回答，不进入正典层。
