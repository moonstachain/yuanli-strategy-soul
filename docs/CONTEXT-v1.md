
---

## 1. 母公理

```text
ʸx：做会自我繁殖的事，不做一次性的事。
```

任何任务都必须问：

```text
这个输出能否成为下一次输入？
这个任务能否让下一轮更强？
这个页面能否指导判断和行动？
这个工具能否被执行、验证、回写、复用？
```

---

## 2. 项目本质

```text
原力战略三部曲不是概念课，而是面向 AI 时代企业家、专家型创业者、高认知个体的个人与事业复利系统。
```

一句话：

```text
找到你的原力母体，把它放进时代与市场，再用 OS 让它自我繁殖。
```

---

## 3. 正典结构

```text
第一部：原力资产
- 发现母体
- 回到母体
- 获得原力
- 显化原力

第二部：原力创业
- 原力借势
- 品类独创
- 模式升维
- 壁垒锁定

第三部：原力OS
- 一纸文脉
- 一个大脑
- 一张地图
- 一条链路
```

不可随意新增第五步。递归回写不是第五步，而是贯穿四步的底层机制。

## 6. 审美原则

```text
暗色
暖金 / 玫瑰金
高信息密度
有呼吸感
像高端战略舱
不是普通工具页
不是 PPT 式堆信息
不是孤立子页
不是紫蓝 AI 渐变
不隐藏不确定性
```

第一眼：企业家知道这和自己有关。
第二眼：知道能拿到什么结果。
第三眼：知道从哪里开始。
第四眼：进入模块深潜。

## 7. 战略边界

必须坚持：

```text
正典优先
下游服从宪法
任务先归属再执行
工具必须可回写
页面必须可行动
白板必须可决策
Issue 必须可验收
Codex 任务必须可执行
```

不做：

```text
不直接污染 CONSTITUTION.md
不未经裁决新增顶层结构
不把白板做成思维导图
不把大脑做成资料堆积
不把链路做成待办清单
不把 Done 做成没有回写的完成
```

---

## 8. AI 使用协议

AI / Codex / Agent 每次参与任务前，必须先判断：

```text
1. 本任务属于资产、创业、OS，还是跨三部？
2. 服务哪条脊骨？
3. 输入是什么？
4. 输出是什么？
5. 验收标准是什么？
6. 回写到哪里？
7. 是否会让下一次更强？
```

## 9. 更新机制

本文件在以下情况需要更新：

```text
母公理解释发生变化

页面审美原则发生变化
```

```text

```
