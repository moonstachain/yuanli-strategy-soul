# Canonical Approval Rubric

## Purpose

Canonical Candidate 不能自动进入正典。任何候选学习、模块定义、课程规则或 agent 规则，都必须通过本 Rubric。

## State Flow

```text
candidate -> reviewed -> approved / rejected
```

## Approval Dimensions

| Dimension | Question | Pass Standard |
|---|---|---|
| Mother Axiom | 是否符合“做会自我繁殖的事，不做一次性的事”？ | 能执行、验证、回写、复用 |
| Evidence | 是否有 evidence 文件或可追溯记录？ | 至少 medium，关键正典需 strong |
| Reuse | 是否能成为下一次输入？ | 有 Next Reuse |
| Strategy Fit | 是否符合三部曲结构和模块边界？ | 不破坏 4x3 架构 |
| Teaching Value | 是否能被学员理解、练习和复盘？ | 可转成 outline / case / assignment |
| Risk | 是否可能误导执行或破坏边界？ | 风险已声明并可控 |

## Decision Rules

- 6/6 pass: approved
- 4-5/6 pass: reviewed, remain candidate
- <=3/6 pass: rejected or repair required
