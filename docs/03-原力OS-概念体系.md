# 原力OS · 概念体系 v0.9

> 系统你：把“做事系统”变成会自我繁殖的工作生命体。  
> 本版关键裁决：**一张地图 = 战略地图 / 能力组件地图 / 事业系统地图**。

---

## 1. 一句话定义

```text
原力OS，是把原力母体、创业经验、知识案例、战略地图与行动链路，装配成可持续递归回写、持续进化的工作生命体。
```

核心公式：

```text
原力OS = 一纸文脉 × 一个大脑 × 一张地图 × 一条链路 × 递归回写
```

白话：

```text
懂你 + 记得住 + 会判断 + 能行动 + 每次变强
```

四步：

```text
一纸文脉 → 一个大脑 → 一张地图 → 一条链路
```

递归回写不是第五步，而是贯穿四步的底层机制。

---

## 2. 四模块总表

| 模块 | 本质 | 白话 | 第一性问题 | 终态产物 |
|---|---|---|---|---|
| 一纸文脉 | 灵魂 / 约束 | 懂你 | AI 如何真正懂我？ | 最小正典文件 |
| 一个大脑 | 记忆 / 索引 | 记得住 | 系统应该记住什么？ | 可检索记忆系统 |
| 一张地图 | 战略判断 / 能力组件 | 会判断 | 系统如何判断方向、取舍、优先级和能力缺口？ | 战略地图 / 能力组件地图 |
| 一条链路 | 行动流 / 执行系统 | 能行动 | 系统如何把判断变成可执行流程？ | OSA / SOP / Agent / Codex 链路 |

完备性判断：

```text
没有文脉：AI 不懂你，只能给通用答案。
没有大脑：系统失忆，每次从零开始。
没有地图：系统有材料但不会战略判断。
没有链路：系统会判断但不能行动。
```

# C1 · 一纸文脉

## 一句话定义

```text
一纸文脉，是把我是谁、为什么做、原力母体、长期承诺、品味函数、战略边界和表达风格，写成 AI、团队与未来自己都能理解和调用的最小正典文件。
```

## 不是它

```text
不是个人简介。
不是品牌文案。
不是愿景口号。
不是提示词集合。
不是一次性项目背景。
```

## 第一性问题

```text
AI 如何真正懂我，并持续保持我的判断边界？
```

## 关键边界

一纸文脉不是让 AI “知道一些我的信息”，而是让 AI 在每次判断时知道：

```text
什么像我。
什么不像我。
什么应该做。
什么绝不做。
什么是长期一致的。
什么只是短期噪音。
```

## 原子模块

```text
母体说明
长期承诺
品味函数
战略边界
语言风格
禁止事项
更新机制
```

## 输入

```text
原力母体总卡
品味函数卡
长期承诺卡
创业验证记录
用户语言与反馈
代表性表达样本
反例样本：不像我 / 不该做 / 不该说
```

## 判断标准

```text
AI 是否能据此判断“像我 / 不像我”？
是否能据此拒绝错误机会？
是否能保持长期口径一致？
是否足够短，能被每次任务调用？
是否可维护，而不是写完就死？
```

## 输出

```text
一纸文脉
我的说明书V3
AI使用协议
战略边界清单
风格词典
禁止事项清单
```

## 回写位置

```text
CONSTITUTION.md
OS 文脉卡
个人元AI上下文
README / docs / templates / skills
```

# C2 · 一个大脑

## 一句话定义

```text
一个大脑，是把知识、案例、术语、样例、数据、经验和任务结果，沉淀成可检索、可调用、可版本化、可更新的记忆系统。
```

## 不是它

```text
不是文件夹堆积。
不是资料收藏。
不是知识库幻觉。
不是没有索引的仓库。
不是一堆无法被任务调用的历史材料。
```

## 第一性问题

```text
系统应该记住什么，才能让下一次不从零开始？
```

## 关键边界

一个大脑不是“资料越多越好”，而是让系统在下一次任务中能回答：

```text
我过去怎么判断过？
哪些案例已经验证过？
哪些术语已经有定义？
哪些样例代表高标准？
哪些数据可以作为下一次判断依据？
```

## 原子模块

```text
知识库
案例库
术语库
样例库
数据库
索引层
版本层
```

## 输入

```text
书稿
源稿
项目案例
客户反馈
模板填写稿
市场数据
任务输出
Evolution Note
Skill蒸馏卡
```

## 判断标准

```text
是否可检索？
是否有来源？
是否有版本？
是否能被下一次任务调用？
是否和一纸文脉一致？
是否能区分事实、判断、样例、模板？
```

## 输出

```text
一个大脑
知识索引
案例索引
术语表
样例库
数据记录
版本记录
```

## 回写位置

```text
source/indexes
source/extracts
案例库
术语库
样例库
数据记录
```

# C3 · 一张地图

## 一句话定义

```text
一张地图，是把文脉、大脑、市场验证和创业结果，组织成一张可判断方向、取舍、优先级、能力缺口和行动路径的战略地图 / 能力组件地图 / 事业系统地图。
```

## 不是它

```text
不是思维导图。
不是信息可视化。
不是漂亮但不能决策的图。
不是战略口号。
不是只罗列模块的架构图。
```

## 第一性问题

```text
系统如何知道什么更重要、先做什么、如何取舍、能力缺口在哪里？
```

## 关键裁决

```text
一张地图 = 战略地图 / 能力组件地图 / 事业系统地图。
```

它接近 IBM CBM 的精神，但用于“原力OS”：把一个人的事业系统拆成可判断、可取舍、可配置、可执行、可升级的能力组件。

## 为什么不是普通地图

普通地图回答：

```text
这里有什么？
```

原力OS的一张地图回答：

```text
什么最重要？
什么是战略控制点？
什么是支撑能力？
什么是瓶颈能力？
什么可外包？
什么必须自己掌握？
什么先做？什么不做？
```

## 原子模块

```text
目标地图
用户地图
机会地图
能力组件地图
控制点地图
因果地图
商业地图
决策阈值
```

## 输入

```text
一纸文脉
一个大脑
原力资产验证记录
原力创业四关结果
用户与市场反馈
客户成功案例
战略目标
任务复盘
```

## 判断标准

```text
是否能帮助取舍？
是否能识别战略控制点？
是否能识别能力缺口？
是否能解释因果？
是否能指导下一步行动？
是否能被验证和更新？
```

## 输出

```text
一张战略地图
能力组件地图
事业系统地图
机会地图
商业地图
变量因果图
行动阈值
战略路径图
```

## 回写位置

```text
OS地图
项目看板
Codex任务树
战略路径图
docs/trilogy-concept-map.html
docs/os.html
```

# C4 · 一条链路

## 一句话定义

```text
一条链路，是把战略地图中的判断，转化为可重复执行、可交给人或AI、可验收、可回写、可自动化的行动流程。
```

## 不是它

```text
不是待办事项。
不是临时任务。
不是一次性交付。
不是自动化脚本堆砌。
不是没有验收标准的流程。
```

## 第一性问题

```text
如何把战略判断变成可复用、可执行、可自动化的行动？
```

## 关键边界

一条链路不是“把事情做完”，而是让每一次行动都能留下：

```text
输入
输出
验收标准
角色分工
工具调用
回写位置
下一次如何更强
```

## 原子模块

```text
OSA任务链
SOP链
Agent链
Codex链
内容链
交付链
回写链
Skill蒸馏链：case → pattern → skill
```

## 输入

```text
一张战略地图
任务目标
工具能力
模板
验收标准
历史流程
人机分工
```

## 判断标准

```text
是否能重复执行？
是否能交给别人或AI？
是否有明确输入输出？
是否有验收标准？
是否有回写位置？
是否能沉淀成 SOP、Agent 或 Skill？
```

## 输出

```text
一条链路
SOP
Agent workflow
Codex-ready task
自动化流程
Evolution Note
Skill蒸馏卡
```

## 回写位置

```text
OS链路
codex/
project/
templates/
skills
原力母体升级记录
```

## 3. 运行层：递归回写

四模块只是器官，让它活起来的是递归回写。

```text
任务 → 输出 → 复盘 → 模式 → 技能 → 回写四组件 → 下一次任务更强
```

回写规则：

| 工作动作 | 回写到哪里 |
|---|---|
| 新洞察 / 新边界 | 一纸文脉 |
| 新知识 / 新案例 / 新样例 | 一个大脑 |
| 新因果 / 新判断 / 新能力缺口 | 一张地图 |
| 新 SOP / Prompt / Agent / Workflow | 一条链路 |
| 可复用能力 | skill 库 |

---

## 4. 三法印

| 三法印 | 约束 | 不满足会怎样 |
|---|---|---|
| 递归进化 | 每次工作必须留下可复用资产 | 系统越用越累，无法复利 |
| 技能统帅 | 先复用 skill，再调用工具，再新增能力 | 重复造轮子，系统失忆 |
| 人类友好 | 低打扰、可解释、保留人的最终判断 | 自动化变成异化，用户放弃 |

优先级：

```text
人类友好 > 递归进化 > 技能统帅
```

---

## 5. 最高验收

一个任务完成后，必须问：

```text
它是否更新了一纸文脉？
它是否沉淀进一个大脑？
它是否更新了一张战略地图？
它是否强化了一条链路？
它是否让下一次更强？
```

若五个答案都是否，它只是一次性劳动，不属于原力OS。
