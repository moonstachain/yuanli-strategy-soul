# 原力OS · 概念体系重构版

> 系统你：把“做事系统”变成会自我繁殖的工作生命体。

---

## 1. 一句话定义

**原力OS，是一个由文脉、大脑、地图、链路组成的工作生命体；它在三法印约束下，把每一次任务递归回写成可复用能力，最终让人从 AI 使用者进化为 ʸx 孵化者。**

核心公式：

```text
原力OS = 文脉 × 大脑 × 地图 × 链路 × 递归回写
```

白话：

```text
懂你 + 记得住 + 会判断 + 能动手 + 每次变强
```

---

## 2. 本体层：四组件

四组件是原力OS的第一主轴。

| 组件 | 白话 | 本质 | 回答的问题 | 对应产物 |
|---|---|---|---|---|
| 文脉 | 灵魂 | 你是谁、为什么做、边界是什么 | 我是谁？ | 一纸文脉 / 我的说明书 |
| 大脑 | 记忆 | 领域知识、案例、经验、判断沉淀 | 我知道什么？ | 知识库 / 企业大脑 |
| 地图 | 判断 | 因果图、战略图、关键战役、决策结构 | 我怎么判断？ | 战略地图 / 决策座舱 |
| 链路 | 手脚 | OSA、agent、workflow、自动化执行 | 我怎么行动？ | 自动化链路 / Agent 流水线 |

完备性判断：

```text
没有文脉：AI 不懂你
没有大脑：系统没记忆
没有地图：系统不会判断
没有链路：系统不能行动
```

---

## 3. 运行层：递归回写

四组件只是器官，让它活起来的是递归回写。

```text
任务 → 输出 → 复盘 → 模式 → 技能 → 回写四组件 → 下一次任务更强
```

回写规则：

| 工作动作 | 回写到哪里 |
|---|---|
| 新洞察 | 文脉 |
| 新资料 / 新案例 | 大脑 |
| 新因果关系 / 新判断 | 地图 |
| 新 SOP / 新 prompt / 新 agent | 链路 |

生命循环：

```text
文脉发起方向
→ 大脑提供材料
→ 地图做出判断
→ 链路执行动作
→ 结果回写文脉/大脑/地图/链路
→ 下一轮更强
```

---

## 4. 宪法层：三法印

三法印不再作为普通章节，而是 OS 的底层约束。

| 三法印 | 约束 | 不满足会怎样 |
|---|---|---|
| 递归进化 | 每次工作必须留下可复用资产 | 系统越用越累，无法复利 |
| 技能统帅 | 先复用 skill，再调用工具，再新增能力 | 重复造轮子，系统失忆 |
| 人类友好 | 低打扰、可解释、保留人的最终判断 | 自动化变成异化，用户放弃 |

---

## 5. 成熟层：S路径

S路径不是另一套目录，而是四组件的成熟度模型。

| 阶段 | 数学量级 | 身份 | 四组件状态 |
|---|---|---|---|
| S1 建模 | 1·x | 学生 | 有初始文脉，会用 AI 建模一个问题 |
| S2 工具 | x·y | 工匠 | 有工具库、大脑库，能复用能力 |
| S3 板 | x^y | 产品人 | 有地图、看板、反馈，系统能自反馈 |
| S4 舱 | ʸx | 孵化者 | 有多链路、多 agent、多系统，可孵化下一代系统 |

---

## 6. 教学层：四部

原来的四部保留，但降级为读者转化路径。

| 四部 | 新定位 |
|---|---|
| 信念 | 让读者相信：必须从“人用工具”切换到“系统自繁殖” |
| 图景 | 让读者看见：OS 长成文脉、大脑、地图、链路 |
| 工具 | 让读者动手：把四组件逐个造出来 |
| 身份 | 让读者完成跃迁：从使用者变成 S4 孵化者 |

---

## 7. 与前两部的咬合

| 来自 | 接入 OS 的方式 |
|---|---|
| 原力资产 | 我的说明书 / 母体定义卡 → 文脉 |
| 原力创业 | 关键战役 / 商业模式 / 用户洞察 → 地图 |
| 原力创业 | SOP / 交付流程 / 复盘模式 → 链路 |
| 全体系 | 所有知识、案例、判断、经验 → 大脑 |

---

## 8. OS 的最高验收

一个任务完成后，必须问：

```text
它是否回写了文脉？
它是否沉淀进大脑？
它是否更新了地图？
它是否强化了链路？
它是否让下一次更强？
```

若五个答案都是否，它只是一次性劳动，不属于原力OS。
