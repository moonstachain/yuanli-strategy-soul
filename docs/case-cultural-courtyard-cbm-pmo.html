01 · 外显原则

这个页面的第一性任务，是让外部用户看懂方法论如何工作，而不是曝光客户细节。

Visibility: public_sanitized_only
它是什么
一个 CBM-PMO Case Candidate 驾驶舱，用来展示战略结构、能力组件、C3/C4、证据等级和人工闸门。
它不是什么
不是客户案例全文，不是会议纪要，不是白板复刻，不是“标杆已完成”的营销页面。
硬边界
任何原始材料、实名信息、收入数据、人员安排、客户隐私均不得进入 public repo。

02 · 三年战略屋

将复杂业务从“项目散点”收束成三年战略、三大抓手和一个中台。

1-2Y cases · 3-5Y scale

北极星：从文化场域到可复用战略操作系统

用产品创新、内容资产化、私域 LTV 与 CDP-PMO-智库中台，形成可复制的学员标杆企业案例。

A · 文化场域产品创新

  • 从传统场景抽取文化母体
  • 重新定义为可礼品化、可私域化的新产品
  • 验证产品、供应链、平台 IT 与私域协同

B · 季节性礼品项目

  • 年度节点型 to C 项目
  • 作为中后台练兵项目
  • 验证研发、供应链、运营衔接丝滑度

C · IP 内容资产 AI 变现

  • 提升既有内容资产激活率
  • 验证 AI 内容工程与增量收益模式
  • 形成可复制给其他 IP 的增长引擎
底座：CDP + PMO + 智库
CDP 管客户与 LTV，PMO 管项目与证据，智库管知识、案例、术语、样例与内容资产。所有后续业务可以新增模块，但不能不继承中台框架。

07 · Human Gate

所有公开、教学、脱敏、数据使用，都必须通过人工闸门。

approval required
Public Naming Gate
是否允许持续使用“文化小院”作为 public alias。
Teaching Material Gate
是否允许进入公开教学材料，哪些部分只能内部使用。
Data Boundary Gate
哪些业务数据可脱敏呈现，哪些永不公开。
Private Archive Gate
是否建立 internal/private 版本承载原始会议纪要与白板图。

08 · 从候选到标杆的路线图

这页只是入口，真正的标杆来自闭环。

M1.5 → M5
M1.5 · Case Candidate

战略白板、会议摘要、项目组合、CBM-PMO v0.1 已形成。当前状态。

M2 · PMO Operating

进入 PMO 看板，C3/C4 有 owner、里程碑、Evidence 路径和 Human Gate。

M3 · First Closed Loop

文化产品、季节性礼品或 AI 内容工程至少一个项目跑完真实闭环。

M4 · Reusable Template

沉淀为可复制模板，可用于其他学员企业或类似文化机构。

M5 · Public Benchmark

脱敏边界通过，人类审批通过，Evidence ≥ E3/E4，成为公开标杆案例。
