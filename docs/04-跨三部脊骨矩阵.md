# 跨三部脊骨矩阵

> 三部曲不是三本散书，而是同一条母公理在“人、事、法”三面墙上的投影。脊骨用于保证所有衍生任务不脱节。

---

## 1. 六条共享脊骨

| 脊骨 | 作用 | 原力资产 | 原力创业 | 原力OS |
|---|---|---|---|---|
| **ʸx** | 母公理 | 资产要会自我繁殖 | 生意要形成财富飞轮 | 系统要孵化下一代智能 |
| **你 × AI** | 主根种子 | 第一性公式 | 关一获得物 | 文脉与元AI的源头 |
| **借势 × 合力** | 财富形态 | 合力来自原型力与天才区 | 创业核心公式 | 提升执行杠杆与系统化能力 |
| **四关** | 市场动作链 | 觉醒阶段提供母体 | 创业主轴 | 作为商业母语复用 |
| **信息五层 / OSA** | 判断底盘 | 找到“你是谁”的全局最优 | 找到打法的全局最优 | 形成地图与链路 |
| **递归回写** | 生命循环 | 复盘自己，升级说明书 | 复盘生意，升级模型 | 复盘任务，升级四组件 |

---

## 2. 任务归属判断

每个任务必须先判断：

```text
它主要服务哪条脊骨？
它主要落在哪一部？
它是否需要跨三部协同？
它的回写位置在哪里？
```

---

## 3. 脊骨映射规则

### 3.1 ʸx

凡是一次性、不可沉淀、不可复用的任务，不进入核心任务池。

### 3.2 你 × AI

所有涉及 IP、个人说明书、天才区、元AI上下文、原型解码的任务，优先归属原力资产。

### 3.3 借势 × 合力

所有涉及用户、市场、品类、商业模式、利润、壁垒的任务，优先归属原力创业。

### 3.4 四关

所有业务衍生任务要标注属于哪一关：

```text
觉醒 / 独创 / 升维 / 锁定
```

### 3.5 信息五层 / OSA

所有研究、判断、分析、决策任务必须经过：

```text
O：目标
S：策略
A：行动
```

并尽量完成信息五层：

```text
碎片 → 特征 → 观点 → 洞察 → 框架 / 全局最优
```

### 3.6 递归回写

每个任务结束时，必须说明回写到哪里：

```text
文脉 / 大脑 / 地图 / 链路 / 看板 / 模板 / 正典文档
```
