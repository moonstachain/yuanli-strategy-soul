# 03a · 原力OS四模块正典重构 v0.9

> 来源：基于 `CONSTITUTION.md`、`docs/03-原力OS-概念体系.md`、`source/extracts/03-原力OS-萃取.md`、`manuscript/03-原力OS-概念与实操.md`、`source/extracts/04d-四四四原子级正典体系-v0.8.md` 的再盘点。  
> 目标：让《原力OS》的四个关键模块，达到《原力资产》《原力创业》同等级别的清晰定义。  
> 本轮关键裁决：**一张地图 = 战略地图 / 能力组件地图 / 事业系统地图**。

---

## 0. 本轮总判断

仓库现有资料已经明确：

```text
原力OS = 文脉 × 大脑 × 地图 × 链路 × 递归回写
```

以及白话：

```text
懂你 + 记得住 + 会判断 + 能行动 + 每次变强
```

但旧口径中，四模块仍有三处不够清晰：

```text
1. “文脉”容易被误解成提示词或品牌文案。
2. “大脑”容易被误解成资料库或文件夹。
3. “地图”容易被误解成思维导图、决策图、信息图。
4. “链路”容易被误解成待办清单或自动化脚本。
```

v0.9 的任务，是把四模块全部校准为可构建、可验收、可回写的 OS 构件。

---

## 1. 原力OS总定义

```text
原力OS，是把原力母体、创业经验、知识案例、战略地图与行动链路，装配成可持续递归回写、持续进化的工作生命体。
```

四步：

```text
一纸文脉 → 一个大脑 → 一张地图 → 一条链路
```

递归回写不是第五步，而是贯穿四步的底层机制。

---

## 2. 四模块总表

| 模块 | 本质 | 白话 | 解决的问题 | 终态产物 |
|---|---|---|---|---|
| 一纸文脉 | 灵魂 / 约束 | 懂你 | AI 如何真正懂我？ | 最小正典文件 |
| 一个大脑 | 记忆 / 索引 | 记得住 | 系统应该记住什么？ | 可检索记忆系统 |
| 一张地图 | 战略判断 / 能力组件 | 会判断 | 系统如何判断方向、取舍、优先级和路径？ | 战略地图 / 能力组件地图 |
| 一条链路 | 行动流 / 执行系统 | 能行动 | 系统如何把判断变成可执行流程？ | OSA / SOP / Agent / Codex 链路 |

# C1 · 一纸文脉

## 一句话定义

```text
一纸文脉，是把我是谁、为什么做、原力母体、长期承诺、品味函数、战略边界和表达风格，写成 AI、团队与未来自己都能理解和调用的最小正典文件。
```

## 不是它

```text
不是个人简介。
不是品牌文案。
不是愿景口号。
不是提示词集合。
不是一次性项目背景。
```

## 第一性问题

```text
AI 如何真正懂我，并持续保持我的判断边界？
```

## 关键边界

一纸文脉不是让 AI “知道一些我的信息”，而是让 AI 在每次判断时知道：

```text
什么像我。
什么不像我。
什么应该做。
什么绝不做。
什么是长期一致的。
什么只是短期噪音。
```

## 原子模块

```text
C1.1 母体说明：我的原力母体是什么。
C1.2 长期承诺：我愿意用十年服务什么。
C1.3 品味函数：什么值得做，什么不值得做。
C1.4 战略边界：不做什么、不服务谁、不用什么方式赚钱。
C1.5 语言风格：常用概念、隐喻、表达方式、审美调性。
C1.6 禁止事项：AI、团队、项目不得越界的内容。
C1.7 更新机制：什么时候必须修订文脉。
```

## 输入

```text
原力母体总卡
品味函数卡
长期承诺卡
创业验证记录
用户语言与反馈
代表性表达样本
反例样本：不像我 / 不该做 / 不该说
```

## 判断标准

```text
AI 是否能据此判断“像我 / 不像我”？
是否能据此拒绝错误机会？
是否能保持长期口径一致？
是否足够短，能被每次任务调用？
是否可维护，而不是写完就死？
```

## 输出

```text
一纸文脉
我的说明书V3
AI使用协议
战略边界清单
风格词典
禁止事项清单
```

## 回写位置

```text
CONSTITUTION.md
OS 文脉卡
个人元AI上下文
README / docs / templates / skills
```

## 完成标志

```text
AI 能稳定用它做判断，而不是只把它当背景材料。
```

# C2 · 一个大脑

## 一句话定义

```text
一个大脑，是把知识、案例、术语、样例、数据、经验和任务结果，沉淀成可检索、可调用、可版本化、可更新的记忆系统。
```

## 不是它

```text
不是文件夹堆积。
不是资料收藏。
不是知识库幻觉。
不是没有索引的仓库。
不是一堆无法被任务调用的历史材料。
```

## 第一性问题

```text
系统应该记住什么，才能让下一次不从零开始？
```

## 关键边界

一个大脑不是“资料越多越好”，而是让系统在下一次任务中能回答：

```text
我过去怎么判断过？
哪些案例已经验证过？
哪些术语已经有定义？
哪些样例代表高标准？
哪些数据可以作为下一次判断依据？
```

## 原子模块

```text
C2.1 知识库：理论、资料、研究、书稿。
C2.2 案例库：客户、项目、交付、成功、失败。
C2.3 术语库：原力体系核心概念与定义。
C2.4 样例库：好输出、坏输出、像我、不像我。
C2.5 数据库：指标、反馈、运营数据、市场证据。
C2.6 索引层：让人和 AI 知道从哪里找。
C2.7 版本层：知道每份知识当前是否仍有效。
```

## 输入

```text
书稿
源稿
项目案例
客户反馈
模板填写稿
市场数据
任务输出
Evolution Note
Skill蒸馏卡
```

## 判断标准

```text
是否可检索？
是否有来源？
是否有版本？
是否能被下一次任务调用？
是否和一纸文脉一致？
是否能区分事实、判断、样例、模板？
```

## 输出

```text
一个大脑
知识索引
案例索引
术语表
样例库
数据记录
版本记录
```

## 回写位置

```text
source/indexes
source/extracts
案例库
术语库
样例库
数据记录
```

## 完成标志

```text
下一次任务能先 search 到旧知识，而不是重新问一遍、重新做一遍。
```

# C3 · 一张地图

## 一句话定义

```text
一张地图，是把文脉、大脑、市场验证和创业结果，组织成一张可判断方向、取舍、优先级、能力缺口和行动路径的战略地图 / 能力组件地图 / 事业系统地图。
```

## 不是它

```text
不是思维导图。
不是信息可视化。
不是漂亮但不能决策的图。
不是战略口号。
不是只罗列模块的架构图。
```

## 第一性问题

```text
系统如何知道什么更重要、先做什么、如何取舍、能力缺口在哪里？
```

## 关键裁决

```text
一张地图 = 战略地图 / 能力组件地图 / 事业系统地图。
```

它接近 IBM CBM 的精神，但用于“原力OS”：把一个人的事业系统拆成可判断、可取舍、可配置、可执行、可升级的能力组件。

## 为什么不是普通地图

普通地图回答：

```text
这里有什么？
```

原力OS的一张地图回答：

```text
什么最重要？
什么是战略控制点？
什么是支撑能力？
什么是瓶颈能力？
什么可外包？
什么必须自己掌握？
什么先做？什么不做？
```

## 原子模块

```text
C3.1 目标地图：当前最重要目标是什么。
C3.2 用户地图：服务谁、不服务谁、心理账户是什么。
C3.3 机会地图：母体与时代势能如何匹配。
C3.4 能力组件地图：事业系统由哪些能力组件构成。
C3.5 控制点地图：哪些组件决定胜负，哪些只是支撑。
C3.6 因果地图：关键变量如何互相影响。
C3.7 商业地图：前链路、后链路、财链路、壁垒在哪里。
C3.8 决策阈值：什么条件下推进、暂停、放弃。
```

## 输入

```text
一纸文脉
一个大脑
原力资产验证记录
原力创业四关结果
用户与市场反馈
客户成功案例
战略目标
任务复盘
```

## 判断标准

```text
是否能帮助取舍？
是否能识别战略控制点？
是否能识别能力缺口？
是否能解释因果？
是否能指导下一步行动？
是否能被验证和更新？
```

## 输出

```text
一张战略地图
能力组件地图
事业系统地图
机会地图
商业地图
变量因果图
行动阈值
战略路径图
```

## 回写位置

```text
OS地图
项目看板
Codex任务树
战略路径图
docs/trilogy-concept-map.html
docs/os.html
```

## 完成标志

```text
它能让系统知道“先做什么、不做什么、哪里是瓶颈、哪里是控制点”。
```

# C4 · 一条链路

## 一句话定义

```text
一条链路，是把战略地图中的判断，转化为可重复执行、可交给人或AI、可验收、可回写、可自动化的行动流程。
```

## 不是它

```text
不是待办事项。
不是临时任务。
不是一次性交付。
不是自动化脚本堆砌。
不是没有验收标准的流程。
```

## 第一性问题

```text
如何把战略判断变成可复用、可执行、可自动化的行动？
```

## 关键边界

一条链路不是“把事情做完”，而是让每一次行动都能留下：

```text
输入
输出
验收标准
角色分工
工具调用
回写位置
下一次如何更强
```

## 原子模块

```text
C4.1 OSA任务链：目标、策略、行动。
C4.2 SOP链：固定流程与验收标准。
C4.3 Agent链：可交给AI执行的任务流。
C4.4 Codex链：输入、输出、文件、验收、禁止事项、回写位置。
C4.5 内容链：选题、生产、分发、复盘。
C4.6 交付链：客户、方案、执行、验收、复购。
C4.7 回写链：任务结束后写回文脉、大脑、地图、链路与母体。
C4.8 Skill蒸馏链：case → pattern → skill。
```

## 输入

```text
一张战略地图
任务目标
工具能力
模板
验收标准
历史流程
人机分工
```

## 判断标准

```text
是否能重复执行？
是否能交给别人或AI？
是否有明确输入输出？
是否有验收标准？
是否有回写位置？
是否能沉淀成 SOP、Agent 或 Skill？
```

## 输出

```text
一条链路
SOP
Agent workflow
Codex-ready task
自动化流程
Evolution Note
Skill蒸馏卡
```

## 回写位置

```text
OS链路
codex/
project/
templates/
skills
原力母体升级记录
```

## 完成标志

```text
它能让同类任务下次更快、更准、更少依赖人肉重来。
```

## 3. 四模块递进关系

```text
一纸文脉：规定系统的灵魂与边界。
一个大脑：提供系统的记忆与材料。
一张地图：做出系统的战略判断与能力配置。
一条链路：把判断变成可执行、可验收、可回写的行动。
```

递进顺序不可颠倒：

```text
没有文脉，大脑会变成杂物箱。
没有大脑，地图会变成空想图。
没有地图，链路会变成瞎执行。
没有链路，地图只能停留在认知层。
```

## 4. 与前两部接口

### 4.1 原力资产 → 原力OS

```text
原力母体 → 一纸文脉
注意力雷达 / 品味函数 / 长期承诺 → 文脉边界
原力能力包 → 大脑样例库
市场验证记录 → 地图与大脑
```

### 4.2 原力创业 → 原力OS

```text
势能地图 → 一张地图
新品类定义 → 一纸文脉 / 大脑术语库
利润容器 → 商业地图 / 链路
财富飞轮 → 能力组件地图 / 壁垒地图
交付SOP → 一条链路
客户成功案例 → 一个大脑
```

### 4.3 原力OS → 原力资产 / 原力创业

```text
文脉回写 → 母体更清楚
大脑回写 → 经验不丢失
地图回写 → 下一轮判断更准
链路回写 → 下一轮执行更快
```
