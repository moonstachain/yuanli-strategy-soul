# 04d · 原力战略三部曲四四四原子级正典体系 v0.8

> Issue: #21 `[正典重构] 四四四原子级重写原力战略三部曲 v0.8`  
> 状态：v0.8 四四四原子级正典稿  
> 目标：基于“原力资产四步 / 原力创业四步 / 原力OS四步”，重新写完整个原力战略三部曲体系。  
> 标准：究竟 / 完备 / 递归 / 具体到原子级。

---

## 0. 本轮总裁决

v0.7 解决了体系是否完备，但仍偏“结构地图”。

v0.8 的任务，是把三部曲升级成更清爽、更正交、更可传播、更可执行的正典骨架：

```text
原力战略三部曲
│
├─ 第一部：原力资产
│   ├─ 发现母体
│   ├─ 回到母体
│   ├─ 获得原力
│   └─ 显化原力
│
├─ 第二部：原力创业
│   ├─ 原力借势
│   ├─ 品类独创
│   ├─ 模式升维
│   └─ 壁垒锁定
│
└─ 第三部：原力OS
    ├─ 一纸文脉
    ├─ 一个大脑
    ├─ 一张地图
    └─ 一条链路
```

这套结构的优势：

```text
1. 三部都是四步，传播更稳。
2. 第一部从“七阶资产机制”压缩成四个更高阶动作。
3. 第二部保留成熟四关。
4. 第三部从组件名升级为可感知的“一纸 / 一个 / 一张 / 一条”。
5. 递归回写不再作为第五步，而是贯穿三部十二步的底层机制。
```

一句话：

```text
找到母体，获得原力；
借势创业，锁定壁垒；
写成文脉，装成系统。
```

---

## 1. 母公理

整套三部曲仍只有一条母公理：

```text
ʸx：做会自我繁殖的事，不做一次性的事。
```

v0.8 对母公理的解释：

```text
任何概念，若不能被执行、验证、回写、复用，就不得进入正典。
任何输出，若不能成为下一次的输入，就不是原力资产。
任何系统，若不能让下一轮更强，就不是原力OS。
```

---

## 2. 三部曲总定义

长定义：

```text
原力战略三部曲，是一套以 ʸx 为母公理，把一个人的原力母体发现、回归、激活、显化，并进一步借势创业、锁定壁垒，最终写入文脉、大脑、地图与链路，形成可递归进化的个人与事业复利系统。
```

短定义：

```text
原力战略三部曲 = 找到母体，借势创业，系统回写。
```

最短定义：

```text
母体 → 原力 → 创业 → OS → 母体升级
```

# 第一部：原力资产

## 4. 原力资产总定义

```text
原力资产，是一个人从发现原力母体开始，重新回到母体，并把母体激活为可调用、可表达、可验证、可回写的原力，最终形成能够进入市场与OS的生命资产。
```

主公式：

```text
原力资产 = 原力母体 × 原力激活 × 真实显化 × OS回写
```

与 v0.6 / v0.7 的关系：

```text
原力母体 × AI外化 × 市场验证 × OS回写
```

在 v0.8 中被归纳为四步：

```text
发现母体 → 回到母体 → 获得原力 → 显化原力
```

这里的“显化原力”不是完整商业化。

裁决：

```text
显化原力 = 市场验证 / 创业接口。
完整变现 = 交给第二部《原力创业》。
```

## A1 · 发现母体

### 一句话定义

```text
发现母体，是从同质化能力、外部标签和二手共识中，识别出自己持续生成判断、作品、关系和意义的底层生命结构。
```

### 不是它

```text
不是兴趣爱好。
不是人格标签。
不是职业优势。
不是人设包装。
不是短期灵感。
```

### 第一性问题

```text
AI 清算所有同质化能力之后，我身上还剩什么会持续生成差异？
```

### 原子模块

```text
A1.1 同质化清算：哪些能力正在被 AI 平权、复制、自动化。
A1.2 异常注意：我天然会注意什么，别人看不见什么。
A1.3 高能事件：什么事情让我越做越清楚、越做越有能量。
A1.4 低级答案厌恶：我最无法忍受哪些粗糙、伪高级或错误答案。
A1.5 他人获得物：别人反复从我这里获得什么。
A1.6 母题浮现：这些现象背后共同指向的主问题是什么。
```

### 输入

```text
过去三年的高能事件
过去三年的低能事件
别人反复找我解决的问题
我长期关注的主题
我最不能忍受的低级答案
AI 已经可以替代的工作清单
```

### 判断标准

```text
它是否长期稳定？
它是否不依赖单一技能？
它是否解释了我过去多个选择？
它是否持续生成判断、作品、关系或意义？
它是否是 AI 难以直接复制的生命结构？
```

### 输出

```text

母题清单
同质化能力清单
母体一句话假设
```

### 下一步接口

```text

```

### 回写位置

```text

我的说明书
OS 文脉

```

```text

不是自我感动。
不是排斥市场。
不是只做喜欢的事。
不是逃避商业化。
```

### 第一性问题

```text
如果我的人生、事业和AI系统都以母体为中心，哪些事情要保留，哪些事情要停止？
```

### 原子模块

```text
A2.4 生命经验：我被哪些经历不可逆地塑形。
A2.5 长期承诺：我愿意用十年持续服务什么。
A2.6 边界与弃绝：哪些机会、关系、任务不再服务母体。
```

# 第二部：原力创业

## 5. 原力创业总定义

```text
原力创业，是把已经显化并初步验证的原力，放进时代势能、用户心智、商业模式和系统壁垒中，形成会复利、守得住、可放大的生意。
```

主公式：

```text
财富 = 借势 × 合力
```

其中：

```text
合力 = 已被发现、回归、激活、显化的原力。
借势 = 时代、结构、入口与母体的同向共振。
```

四步：

```text
原力借势 → 品类独创 → 模式升维 → 壁垒锁定
```

---

## B1 · 原力借势

### 一句话定义

```text
原力借势，是判断这个时代为什么现在需要我的原力，并找到最适合进入市场的势能窗口。
```

### 不是它

```text
不是追风口。
不是蹭热点。
不是宏观叙事自嗨。
不是把母体强行塞进不适合的趋势。
```

### 第一性问题

```text
这个时代为什么现在需要我？
```

### 原子模块

```text
B1.1 宏周期势：康波、技术范式、资本周期、AI渗透率。
B1.2 中结构势：政策、经济、社会、技术、产业价值链。
B1.3 微入口势：用户心智、心理账户、渠道窗口、预算迁移。
B1.4 合力匹配：我的原力是否刚好适配这股势。
B1.5 时间窗口：早、刚好、晚，分别对应不同打法。
B1.6 势能排序：多个机会如何排序。
```

### 输入

```text
创业输入包
市场验证记录
行业趋势资料
用户变化信号
政策 / 技术 / 渠道变化
```

### 判断标准

```text
是否只是热闹风口？
是否存在中长期结构变化？
是否与我的原力同向？
是否有明确入口和预算迁移？
```

### 输出

```text

入场窗口判断
机会优先级
```

# 第三部：原力OS

## 6. 原力OS总定义

```text
原力OS，是把原力母体、创业经验、知识案例、判断地图与行动链路，装配成可持续递归回写、持续进化的工作生命体。
```

主公式：

```text
原力OS = 一纸文脉 × 一个大脑 × 一张地图 × 一条链路 × 递归回写
```

四步：

```text
一纸文脉 → 一个大脑 → 一张地图 → 一条链路
```

裁决：

```text
递归回写不是第五步，而是贯穿四步的底层机制。
```

## C1 · 一纸文脉

### 一句话定义

```text
一纸文脉，是把我是谁、为什么做、原力母体、长期边界和表达风格，写成 AI 与团队都能理解和调用的最小正典文件。
```

### 不是它

```text
不是个人简介。
不是品牌文案。
不是愿景口号。
不是提示词集合。
```

### 第一性问题

```text
AI 如何真正懂我？
```

### 原子模块

```text
C1.1 母体说明：我的原力母体是什么。
C1.2 长期承诺：我愿意用十年服务什么。
C1.3 品味函数：什么值得做，什么不值得做。
C1.4 战略边界：什么不做、服务谁、不服务谁。
C1.5 语言风格：常用概念、隐喻、表达方式、审美偏好。
C1.6 禁止事项：AI 与团队不得越界的内容。
```

### 输入

```text
原力母体总卡
品味函数卡
长期承诺卡
创业验证记录
代表性表达样本
战略边界
```

### 判断标准

```text
AI 是否能基于它稳定判断？
是否知道什么不该做？
是否能保持口径一致？
是否足够短，能被持续维护？
```

### 输出

```text
一纸文脉
我的说明书V3
AI使用协议
战略边界清单
```

### 下一步接口

```text
进入 C2 一个大脑。
```

### 回写位置

```text
CONSTITUTION.md
OS 文脉卡
个人元AI上下文
docs / templates / skills
```

## C4 · 一条链路

### 一句话定义

```text
一条链路，是把地图中的判断转化为可重复执行、可交给人或AI、可验收、可回写的行动流程。
```

### 不是它

```text
不是待办事项。
不是临时任务。
不是一次性交付。
不是没有验收标准的流程。
```

### 第一性问题

```text

```

### 原子模块

```text
C4.1 OSA任务链：目标、策略、行动。
C4.2 SOP链：固定流程与验收标准。
C4.3 Agent链：可交给AI执行的任务流。
C4.4 Codex链：输入、输出、文件、验收、禁止事项、回写位置。
C4.5 内容链：选题、生产、分发、复盘。
C4.6 交付链：客户、方案、执行、验收、复购。
C4.7 回写链：任务结束后写回文脉、大脑、地图、链路与母体。
```

### 判断标准

```text
是否能重复执行？
```
