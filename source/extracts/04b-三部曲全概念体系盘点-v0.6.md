## 0. 本轮总裁决

三部曲当前正典不再以旧的“你 × AI”作为《原力资产》的最终表达，而升级为：

```text
原力资产 = 原力母体 × AI外化 × 市场验证 × OS回写
```

其中：

```text
原力母体 = 注意力雷达 × 品味函数 × 身体化能力 × 生命经验 × 长期承诺
```

三部曲的总飞轮为：

```text
原力资产 → 原力创业 → 原力OS → 原力资产升级
```

压缩成一句话：

```text
看见母体 → 借势母体 → 变现母体 → 系统回写母体 → 升级母体
```

更短：

```text
原力战略三部曲 = 让原力母体进入时代、市场与系统，并持续自我繁殖。
```

## 1. L0：母公理层

### 1.1 母公理

```text
ʸx：做会自我繁殖的事，不做一次性的事。
```

判断标准：

```text
完成一次之后，是否让下一次更容易？
完成一个之后，是否能孵化下一个？
完成一轮之后，是否能回写成资产、能力、系统或判断？
```

### 1.2 母公理对三部曲的约束

```text
原力资产：不是一次性自我理解，而是会复利的原力母体。
原力创业：不是一次性赚钱机会，而是会复利的生意系统。
原力OS：不是一次性工具提效，而是会自我繁殖的工作生命体。
```

### 1.3 母公理对所有任务的约束

任何任务都必须回答：

```text
它会回写到哪里？
它下次如何复用？
它是否会让下一次更强？
```

## 2. L1：三部曲总结构

```text
原力战略三部曲 = 原力资产 / 原力创业 / 原力OS
```

| 部 | 方向 | 对象 | 角色 | 核心问题 | 终态 |
|---|---|---|---|---|---|
| 原力资产 | 向内 | 人 | 找到母体 | 我真正不可复制的源头是什么？ | 会复利的原力母体 |
| 原力创业 | 向外 | 事 | 借势变现 | 这个母体该借哪股势进入市场？ | 会复利的生意 |
| 原力OS | 向上 | 法 | 系统回写 | 如何让每次任务沉淀成下一次能力？ | ʸx 孵化者 |

三部不是并列三本书，而是一只飞轮：

```text
原力资产生成母体 → 原力创业借势变现 → 原力OS递归回写 → 原力资产升级
```

## 3. L2：三部核心公式

### 3.1 原力资产公式

```text
原力资产 = 原力母体 × AI外化 × 市场验证 × OS回写
```

四段解释：

```text
原力母体：我为什么能持续生成差异。
AI外化：这些差异如何被表达、产品化、系统化。
市场验证：这些差异是否真的被人相信、靠近、付费、复用和传播。
OS回写：每一次反馈如何反过来升级我的原力母体。
```

### 3.2 原力创业公式

```text
财富 = 借势 × 合力
```

其中：

```text
借势 = 宏周期势 × 中结构势 × 微入口势 × 合力匹配
合力 = 被识别、外化、验证过的原力母体
```

### 3.3 原力OS公式

```text
原力OS = 文脉 × 大脑 × 地图 × 链路 × 递归回写
```

白话：

```text
懂你 + 记得住 + 会判断 + 能动手 + 每次变强
```

## 4. L3：第一部《原力资产》全概念体系

### 4.1 主定义

```text
原力资产，是一个人以原力母体为源头，经由 AI 外化、市场验证与 OS 回写，形成的可复利、不可复制、可进入时代与市场的生命资产。
```

最短定义：

```text
原力资产 = 会复利的原力母体
```

### 4.2 七阶路径

```text
清算 → 返源 → 成形 → 外化 → 验证 → 回写 → 复利
```

| 阶 | 名称 | 核心问题 | 输出 |
|---|---|---|---|
| 1 | 清算 | 哪些能力正在被 AI 同质化？ | 同质化能力清单 |
| 2 | 返源 | 我的原力母体是什么？ | 原力母体总卡 |
| 3 | 成形 | 母体的五个部件如何结构化？ | 五部件卡组 |
| 4 | 外化 | AI 如何把母体变成可见资产？ | 表达 / 模型 / 产品 / 方法 / 系统 |
| 5 | 验证 | 世界是否真的需要这个母体？ | 天选用户 / 贵问题 / 验证记录 |
| 6 | 回写 | 反馈如何反过来升级母体？ | 回写记录 / 母体更新 |
| 7 | 复利 | 母体如何越用越清楚、越值钱？ | 可进化身份 |

### 4.3 原力母体五部件

```text
注意力雷达：我天然会看见什么，别人看不见什么。
品味函数：我如何判断什么值得做、什么不值得做。
身体化能力：我长期训练出的手感、嗅觉、判断和行动回路。
生命经验：我被哪些经历不可逆地塑形。
长期承诺：我愿意用十年持续服务什么。
```

## 5. L3：第二部《原力创业》全概念体系

### 5.1 主定义

```text
原力创业，是把已经识别、外化并初步验证的原力母体，放进时代窗口、用户心智、商业模式与系统壁垒中，形成会复利的生意。
```

### 5.2 第一性公式

```text
财富 = 借势 × 合力
```

其中：

```text
合力来自原力资产。
借势来自时代、结构、入口与匹配。
```

### 5.3 四关通关结构

```text
原力借势 → 品类独创 → 模式升维 → 壁垒锁定
```

| 关 | 第一性问题 | 获得物 | 失败状态 |
|---|---|---|---|
| 原力借势 | 这个时代为什么现在需要我？ | 势能地图 | 怀才不遇 / 风口炮灰 |
| 品类独创 | 市场如何认得我？ | 心理账户 / 品类入口 | 做得更好但没人记住 |
| 模式升维 | 怎么持续赚钱、可复制？ | 利润容器 | 人肉交付 / 规模即崩 |
| 壁垒锁定 | 怎么守住、不被拿走？ | 财富飞轮 | 先发但被复制 |

### 5.4 原力借势四层

```text
原力借势 = 宏周期势 × 中结构势 × 微入口势 × 合力匹配
```

四层解释：

```text
宏周期势：康波、技术范式、资本周期、AI渗透率。
中结构势：政策、经济、社会、技术、产业价值链。
微入口势：用户心智、心理账户、渠道窗口、预算迁移。
合力匹配：我的原力母体是否刚好适配这股势。
```

### 5.5 品类独创

核心任务：

```text
让市场用一个新名字认得我。
```

关键结构：

```text
甜用户 × 贵任务 × 入脑表达
心理账户
一句话价值主张
新品类命名
```

### 5.6 模式升维

核心任务：

```text
从个人能干，升级为系统能跑。
```

关键结构：

```text
前链路：获客 / 信任 / 内容 / 转化
后链路：交付 / 复购 / 客户成功 / 案例沉淀
财链路：收入结构 / 毛利结构 / 现金流 / 杠杆
```

### 5.7 壁垒锁定

核心任务：

```text
让生意不只是跑起来，而是守得住。
```

关键结构：

```text
虚：认知、品牌、意义、叙事
实：产品、交付、数据、供应链
入：入口、渠道、社群、关系
出：复购、迁移成本、生态、系统
```

## 6. L3：第三部《原力OS》全概念体系

### 6.1 主定义

```text
原力OS，是把人的文脉、知识、判断与行动链路，装配成会递归回写、持续进化的工作生命体。
```

### 6.2 主公式

```text
原力OS = 文脉 × 大脑 × 地图 × 链路 × 递归回写
```

### 6.3 四组件

```text
文脉：灵魂。回答我是谁、为什么做、边界是什么。
大脑：记忆。回答我知道什么、有哪些案例、经验和知识。
地图：判断。回答我如何决策、如何判断优先级、如何看因果。
链路：手脚。回答我如何行动、如何交付、如何自动化。
```

### 6.4 递归回写

```text
任务 → 输出 → 复盘 → 模式 → skill → 回写四组件 → 下一次任务更强
```

### 6.5 数学地基

```text
1·x：你 + 1 个工具
x·y：你 + N 个工具
x^y：你 + N 个工具 + 工具自反馈
ʸx：你 + N 个智能 + 智能造下一代智能
```

### 6.6 三法印

```text
递归进化：每一次任务都让下一次更强。
技能统帅：先 search，再 call；先理解 skill，再调用工具。
人类友好：节省 token、不乱打扰、不夺走人的最终判断权。
```

优先级：

```text
人类友好 > 递归进化 > 技能统帅
```

### 6.7 OSA

```text
O：目标层。我们要干什么？为什么是这件不是那件？
S：策略层。怎么打？哪条路径成功率最高？
A：行动层。谁做？何时做？如何验真？
```

### 6.8 信息五层

```text
碎片 → 特征 → 观点 → 洞察 → 框架
```

### 6.9 Skill 蒸馏

```text
case → pattern → skill
```

### 6.10 S 路径

```text
S1 建模：1·x，学生
S2 工具：x·y，工匠
S3 板：x^y，产品人
S4 舱：ʸx，孵化者
```
