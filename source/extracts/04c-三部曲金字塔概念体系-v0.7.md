## 0. 金字塔原则

本体系以后不再采用“概念堆叠式地图”，而采用金字塔结构：

```text
一级：三部曲
二级：每部主机制
三级：机制步骤
四级：步骤子模块
五级：输入 / 判断 / 输出 / 接口
```

判断一张概念地图是否合格，不看概念多不多，而看：

```text
1. 上层是否统摄下层。
2. 下层是否穷尽上层。
3. 同层是否互斥。
4. 每个模块是否有输入、判断、输出、接口。
5. 整体是否能递归回写。
```

---

## 1. 一级：原力战略三部曲

```text
原力战略三部曲 = 原力资产 / 原力创业 / 原力OS
```

三部曲的递进：

```text
找到母体 → 借势母体 → 系统回写母体
```

三部曲的飞轮：

```text
原力资产 → 原力创业 → 原力OS → 原力资产升级
```

---

## 2. 第二级：三部主机制

| 部 | 主机制 | 主公式 | 结果 |
|---|---|---|---|
| 原力资产 | 资产生成机制 | 原力母体 × AI外化 × 市场验证 × OS回写 | 会复利的原力母体 |
| 原力创业 | 商业通关机制 | 原力借势 → 品类独创 → 模式升维 → 壁垒锁定 | 会复利的生意 |
| 原力OS | 系统回写机制 | 文脉 × 大脑 × 地图 × 链路 × 递归回写 | ʸx 孵化者 |

# 第一部：原力资产

## 3. 原力资产主机制

```text
原力资产 = 原力母体 × AI外化 × 市场验证 × OS回写
```

七阶路径：

```text
清算 → 返源 → 成形 → 外化 → 验证 → 回写 → 复利
```

---

## 3.1 清算

### 定义

清算，是识别哪些能力正在被 AI 同质化，停止把可复制能力误认为长期资产。

### 核心问题

```text
哪些能力正在被 AI 压低价格？
哪些优势只是时代训练、行业范式或二手共识？
```

### 子模块

```text
1. 可学习能力清算：知识、技能、工具使用。
2. 可表达能力清算：写作、总结、讲述、设计。
3. 可执行能力清算：流程、交付、运营、代码。
4. 可模仿风格清算：语言、审美、视觉、内容套路。
5. 可流程化经验清算：SOP、模板、自动化任务。
```

### 输入

```text
过去一年主要工作清单
AI 已能替代的任务
行业通用能力列表
自己以为的优势清单
```

### 判断标准

```text
如果别人用 AI + 模板能在短期内接近，这不是核心资产。
如果能力越普及价格越低，这不是核心资产。
如果离开我的母体仍能被复制，这不是核心资产。
```

### 输出

```text
同质化能力清单
不可误认资产清单
需要外包 / 自动化 / 系统化的任务清单
```

### 下一步接口

进入“返源”：从被清算后的剩余部分寻找原力母体。

---

## 3.2 返源

### 定义

返源，是从技能、身份、标签背后，回到持续生成差异的原力母体。

### 核心问题

```text
我为什么会持续生成这些判断、作品、关系和意义？
```

### 子模块

```text
1. 长期注意力：我反复被什么吸引。
2. 重复判断：我总能在哪些问题上产生判断。
3. 低级答案厌恶：我最无法忍受什么。
4. 他人获得物：别人反复从我这里获得什么。
5. 母题浮现：这些现象背后共同指向什么。
```

### 输入

```text
高能事件
低能事件
被反复求助的问题
长期关注主题
最常输出的观点或作品
```

### 判断标准

```text
它是否长期稳定？
它是否持续生成差异？
它是否不是单一技能？
它是否能解释我过去多个选择？
```

### 输出

```text
原力母体候选
母体一句话假设
需要验证的母题
```

### 下一步接口

进入“成形”：把母体拆成五个可操作部件。

---

## 3.3 成形

### 定义

成形，是把原力母体从隐喻拆成可识别、可填写、可外化、可验证的五个部件。

### 核心问题

```text
我的原力母体由哪五个部件构成？
```

### 子模块

```text
1. 注意力雷达：我天然会看见什么。
2. 品味函数：我如何判断什么值得做。
3. 身体化能力：我长期训练出的手感和行动回路。
4. 生命经验：我被哪些经历不可逆地塑形。
5. 长期承诺：我愿意用十年持续服务什么。
```

### 输入

```text
原力母体候选
注意力事件
品味样本
能力案例
生命经历
长期问题
```

### 判断标准

```text
五部件是否互相支撑？
注意力是否能解释品味？
品味是否能筛选行动？
能力是否有长期反馈来源？
经验是否指向承诺？
```

### 输出

```text
原力母体总卡
注意力雷达卡
品味函数卡
身体化能力卡
生命经验炼金卡
长期承诺卡
```

### 下一步接口

进入“外化”：让 AI 把母体变成可见资产。

---

## 3.4 外化

### 定义

外化，是用 AI 把原力母体从隐性判断变成可见表达、模型、产品、方法和系统。

### 核心问题

```text
AI 如何把我的母体外化为可被看见、使用和传播的资产？
```

### 子模块

```text
1. 语言外化：把隐性判断变成概念和表达。
2. 模型外化：把经验变成框架和判断模型。
3. 产品外化：把能力变成服务、工具、课程或作品。
4. 流程外化：把手感变成 SOP / workflow / agent。
5. 文脉外化：把母体写进我的说明书与 OS 文脉。
```

### 输入

```text
六张资产卡
代表性案例
高质量输出样本
用户反馈
长期承诺方向
```

### 判断标准

```text
AI 是否放大了我的母体，而不是平均答案？
输出是否能让别人更清楚我是谁？
输出是否能被复用、传播或验证？
```

### 输出

```text
母体说明书
内容主题库
方法论模型
产品原型
AI 外化样例
OS 文脉种子
```

### 下一步接口

进入“验证”：把外化产物放进真实世界。

---

## 3.5 验证

### 定义

验证，是确认这个母体是否真的被真实世界需要。

### 核心问题

```text
世界是否真的需要这个母体？
```

### 子模块

```text
1. 靠近验证：是否有人持续靠近。
2. 相信验证：是否有人愿意相信和引用。
3. 付费验证：是否有人愿意付钱。
4. 复用验证：是否有人反复使用。
5. 行动验证：是否有人因此改变行动。
6. 共建验证：是否有人愿意共同建设。
```

### 输入

```text
外化产物
目标人群
内容 / 产品 / 服务原型
真实互动记录
初始成交或复用记录
```

### 判断标准

```text
是否有真实行为，而不是口头喜欢？
是否有持续反馈，而不是一次性热闹？
是否出现天选用户？
是否显现贵问题？
```

### 输出

```text
天选用户线索
贵问题线索
心理账户线索
市场验证记录
进入创业的机会假设
```

### 下一步接口

交给《原力创业》第一关“原力借势”。

---

## 3.6 回写

### 定义

回写，是把市场、身体、关系、任务与 AI 反馈重新写回原力母体。

### 核心问题

```text
这次反馈如何让我更清楚自己的母体？
```

### 子模块

```text
1. 市场回写：谁是真用户，什么是真问题。
2. 身体回写：什么让我高能或消耗。
3. 关系回写：什么关系放大或消耗母体。
4. 任务回写：什么能力需要沉淀或外包。
5. AI回写：说明书、文脉、样例如何更新。
6. OS回写：写入文脉、大脑、地图、链路。
```

### 输入

```text
市场反馈
交付复盘
身体状态
关系反馈
AI 输出偏差
任务结果
```

### 判断标准

```text
反馈是否被结构化？
是否明确回写位置？
是否改变下一次判断或行动？
```

### 输出

```text
母体更新记录
OS 回写记录
新判断标准
新用户假设
新产品假设
```

### 下一步接口

进入“复利”：让母体越用越强。

---

## 3.7 复利

### 定义

复利，是原力母体经过多轮外化、验证、回写之后，形成可进化身份。

### 核心问题

```text
我的母体是否越用越清楚、越用越值钱？
```

### 子模块

```text
1. 母体复利：注意力、品味、能力、经验、承诺更清楚。
2. AI复利：AI 越来越懂我，外化越来越准。
3. 市场复利：用户和贵问题越来越清晰。
4. OS复利：文脉、大脑、地图、链路越来越强。
5. 财富接口：母体开始进入创业与资产化。
```

### 输入

```text
多轮验证记录
母体版本记录
AI 文脉版本
市场反馈曲线
OS 回写日志
```

### 判断标准

```text
下一次是否比上一次更容易？
判断是否更准？
用户是否更清晰？
产出是否更可复用？
```

### 输出

```text
会复利的原力母体
创业输入包
OS 文脉包
下一轮母体升级计划
```

### 下一步接口

正式进入《原力创业》：借势母体。

# 第二部：原力创业

## 4. 原力创业主机制

```text
原力创业 = 原力借势 → 品类独创 → 模式升维 → 壁垒锁定
```

主公式：

```text
财富 = 借势 × 合力
```

其中：

```text
合力 = 被识别、外化、验证过的原力母体。
借势 = 让母体进入时代窗口、用户心智与商业结构。
```

---

## 4.1 原力借势

### 定义

原力借势，是判断这个时代为什么现在需要我的原力母体。

### 核心问题

```text
这个母体该借哪股势？
```

### 子模块

```text
1. 宏周期势：康波、技术范式、AI渗透率、资本周期。
2. 中结构势：政策、经济、社会、技术、产业链。
3. 微入口势：用户心智、心理账户、渠道窗口、预算迁移。
4. 合力匹配：我的母体是否刚好适配这股势。
5. 时间窗口：现在入场是早、刚好，还是晚。
```

### 输入

```text
原力母体输入包
市场验证记录
行业趋势资料
用户变化信号
政策 / 技术 / 渠道变化
```

### 判断标准

```text
是否只是风口？
是否具备中长期结构性变化？
是否和我的母体匹配？
是否存在明确入口和窗口？
```

### 输出

```text
势能地图
机会假设
入场窗口判断
优先级排序
```

### 下一步接口

进入“品类独创”：让市场认得我。

---

## 4.2 品类独创

### 定义

品类独创，是把原力母体翻译成市场能识别、能记住、愿意付费的新入口。

### 核心问题

```text
市场如何认得我？
```

### 子模块

```text
1. 甜用户：谁最自然需要我。
2. 贵任务：什么问题值得付费。
3. 心理账户：用户把钱、时间、注意力放在哪个账户。
4. 旧品类对标：用户原来如何理解这类问题。
5. 新品类命名：我不是旧方案，而是什么新方案。
6. 入脑表达：一句话让人记住、转述、区分。
```

### 输入

```text
势能地图
天选用户线索
贵问题线索
用户语言
竞品 / 旧方案
```

### 判断标准

```text
用户是否一听就懂？
是否能转述？
是否知道为什么付费？
是否能把我和旧方案区分开？
```

### 输出

```text
新品类定义
一句话价值主张
心理账户定位
品类命名候选
初始内容 / 产品表达
```

### 下一步接口

进入“模式升维”：把品类变成利润容器。

---

## 4.3 模式升维

### 定义

模式升维，是把一次性交付升级为可复制、可复利、可扩张的利润容器。

### 核心问题

```text
怎么持续赚钱、可复制？
```

### 子模块

```text
1. 前链路：获客、信任、内容、转化。
2. 后链路：交付、复购、客户成功、案例沉淀。
3. 财链路：收入结构、毛利结构、现金流、杠杆。
4. 产品阶梯：免费内容、小产品、核心产品、高阶交付。
5. 数据反馈：每轮交付如何产生下一轮增长资产。
```

### 输入

```text
品类定义
目标用户
价值主张
交付能力
价格假设
渠道与内容资产
```

### 判断标准

```text
是否一增长就人肉崩溃？
是否有可复用交付？
是否有清晰毛利和现金流？
是否有复购和转介绍机制？
```

### 输出

```text
利润容器
产品阶梯
三链路画布
交付 SOP
增长指标
```

### 下一步接口

进入“壁垒锁定”：让生意守得住。

---

## 4.4 壁垒锁定

### 定义

壁垒锁定，是让已经跑通的生意不被轻易复制、拿走或替代。

### 核心问题

```text
怎么守住、不被拿走？
```

### 子模块

```text
1. 虚壁垒：认知、品牌、意义、叙事、品味。
2. 实壁垒：产品、交付、数据、供应链、工具。
3. 入壁垒：入口、渠道、社群、关系、信任。
4. 出壁垒：复购、迁移成本、生态、系统、身份绑定。
5. 飞轮壁垒：每轮用户、数据、案例、内容反哺下一轮。
```

### 输入

```text
利润容器
客户成功案例
数据资产
品牌表达
渠道资产
交付系统
```

### 判断标准

```text
别人复制需要多久？
复制后是否仍缺我的母体？
用户离开成本是否上升？
系统是否越跑越强？
```

### 输出

```text
财富飞轮
壁垒地图
16控制点
资产护城河
OS 回写包
```

### 下一步接口

交给《原力OS》：把创业经验系统化。

# 第三部：原力OS

## 5. 原力OS主机制

```text
原力OS = 文脉 × 大脑 × 地图 × 链路 × 递归回写
```

OS 的任务：

```text
把一次任务变成下一次能力，
把一次交付变成下一次链路，
把一次判断变成下一次地图，
把一次成功蒸馏成下一次 skill。
```

---

## 5.1 文脉

### 定义

文脉，是 OS 的灵魂，回答“我是谁、为什么做、边界是什么”。

### 核心问题

```text
AI 如何真正理解我？
```

### 子模块

```text
1. 原力母体说明书。
2. 长期愿景与承诺。
3. 价值排序与品味函数。
4. 禁止事项与边界。
5. 表达风格与审美偏好。
```

### 输入

```text
原力母体总卡
品味函数卡
长期承诺卡
关键表达样本
重要边界
```

### 判断标准

```text
AI 是否能稳定像我一样判断？
是否知道什么不该做？
是否能在不同任务中保持一致性？
```

### 输出

```text
文脉卡
我的说明书
AI 使用协议
风格与边界清单
```

### 下一步接口

交给大脑：决定哪些知识和案例需要记住。

---

## 5.2 大脑

### 定义

大脑，是 OS 的记忆，回答“我知道什么”。

### 核心问题

```text
系统应该记住哪些知识、案例、经验和判断？
```

### 子模块

```text
1. 知识库：理论、资料、书稿、研究。
2. 案例库：客户、项目、交付、失败和成功。
3. 术语库：原力体系内部概念定义。
4. 样例库：好输出、坏输出、像我、不像我。
5. 数据库：指标、反馈、运营数据。
```

### 输入

```text
书稿
源稿
项目案例
客户反馈
模板填写稿
市场数据
```

### 判断标准

```text
是否可检索？
是否可信？
是否有版本？
是否能被下一次任务调用？
```

### 输出

```text
大脑库卡
知识索引
案例索引
术语表
样例库
```

### 下一步接口

交给地图：让知识变成判断结构。

---

## 5.3 地图

### 定义

地图，是 OS 的判断系统，回答“我怎么决策”。

### 核心问题

```text
系统如何知道什么更重要、先做什么、怎么判断？
```

### 子模块

```text
1. 因果地图：变量之间如何影响。
2. 战略地图：目标、路径、关键战役。
3. 用户地图：人群、心理账户、贵问题。
4. 商业地图：链路、模式、壁垒。
5. 决策地图：判断标准、阈值、行动条件。
```

### 输入

```text
大脑库资料
市场验证记录
创业四关结果
OS 任务复盘
战略目标
```

### 判断标准

```text
地图是否能帮助取舍？
是否能解释因果？
是否能指导下一步行动？
是否能被更新？
```

### 输出

```text
决策地图卡
变量因果图
机会地图
战略路径图
行动阈值
```

### 下一步接口

交给链路：把判断变成行动流程。

## 5.4 链路

### 定义

链路，是 OS 的手脚，回答“我怎么行动”。

### 核心问题

```text
如何把判断变成可复用、可执行、可自动化的行动？
```

### 子模块

```text
1. OSA 任务链：目标、策略、行动。
2. SOP 链：固定流程与验收标准。
3. Agent 链：可交给 AI 执行的任务流。
4. 内容链：选题、生产、分发、复盘。
5. 交付链：客户、方案、执行、验收、复购。
```

### 输入

```text
决策地图
任务目标
工具能力
模板
验收标准
历史流程
```

### 判断标准

```text
是否能重复执行？
是否能交给别人或 AI？
是否有明确输入输出？
是否有验收标准？
```

### 输出

```text
链路卡
SOP
Agent workflow
Codex-ready task
自动化流程
```

### 下一步接口

进入递归回写。

## 5.5 递归回写

### 定义

递归回写，是让每一次任务反过来增强文脉、大脑、地图和链路。

### 核心问题

```text
这一次任务如何让下一次任务更强？
```

### 子模块

```text
1. Evolution Note：本次任务产生了什么新事实、新洞察、新模式。
2. 信息五层：碎片 → 特征 → 观点 → 洞察 → 框架。
3. Skill 蒸馏：case → pattern → skill。
4. 版本升级：v0.1 → v0.2 → v1.0。
5. 母体回写：任务反馈如何更新原力母体。
```

### 输入

```text
任务输出
复盘记录
用户反馈
失败记录
成功模式
工具调用记录
```

### 判断标准

```text
是否产生新模式？
是否能蒸馏为 skill？
是否明确回写位置？
是否让下一次更快、更准、更好？
```

### 输出

```text
Evolution Note
Skill 蒸馏卡
文脉更新
大脑更新
地图更新
链路更新
母体升级记录
```

### 下一步接口

回到《原力资产》：升级母体；回到《原力创业》：升级生意。

## 5.6 OS 宪法层

### 三法印

```text
递归进化：每一次任务都让下一次更强。
技能统帅：先 search，再 call；先理解 skill，再调用工具。
人类友好：节省 token、不乱打扰、不夺走人的最终判断权。
```

优先级：

```text
人类友好 > 递归进化 > 技能统帅
```

### OSA

```text
O：目标层。我们要干什么？为什么是这件不是那件？
S：策略层。怎么打？哪条路径成功率最高？
A：行动层。谁做？何时做？如何验真？
```

### S 路径

```text
S1 建模：1·x，学生
S2 工具：x·y，工匠
S3 板：x^y，产品人
S4 舱：ʸx，孵化者
```
