# 03 · 原力OS：系统你

## 03.1 为什么有了创业，还需要 OS

《原力资产》解决“找到你”。

《原力创业》解决“借势你、变现你”。

但这还不够。

因为一个人可以找到自己，也可以把自己变成一门生意，但如果每一次判断、每一次交付、每一次写作、每一次客户方案，都只能靠人重新来过，这仍然不是 ʸx。

这只是更高级的人肉劳动。

真正的问题是：

```text
这一次任务，能不能让下一次任务更容易？
这一次交付，能不能变成下一次交付的链路？
这一次判断，能不能沉淀成下一次判断的地图？
这一次成功，能不能蒸馏成下一次可以调用的 skill？
```

这就是《原力OS》的任务。

原力OS不是工具箱，不是提示词合集，也不是 AI 自动化脚本。

它是一个工作生命体。

## 03.2 原力OS = 文脉 × 大脑 × 地图 × 链路 × 递归回写

原力OS的核心公式是：

```text
原力OS = 文脉 × 大脑 × 地图 × 链路 × 递归回写
```

白话版：

```text
懂你 + 记得住 + 会判断 + 能动手 + 每次变强
```

四个组件分别回答四个问题：

```text
文脉：我是谁？
大脑：我知道什么？
地图：我怎么判断？
链路：我怎么行动？
```

递归回写回答第五个问题：

```text
我如何每一次都变强？
```

只有五个都成立，才不是“用 AI 做事”，而是“造出一个会自我生长的工作生命体”。

## 03.3 数学地基：从 x^y 到 ʸx

原力OS的数学地基，是从 x^y 到 ʸx 的跃迁。

四个量级是：

```text
1·x   你 + 1 个工具
x·y   你 + N 个工具
x^y   你 + N 个工具 + 工具自反馈
ʸx    你 + N 个智能 + 智能造下一代智能
```

前三个阶段，主语仍然是你。

你在使用工具，你在调度团队，你在推动飞轮。

到了 ʸx，主语变了。

系统开始自己生系统，智能开始造下一代智能，skill 开始从 case 中自动蒸馏，agent 开始把经验回写为下一次能力。

所以原力OS不是为了“提效10倍”。

提效10倍，仍然可能只是 x·y。

原力OS要完成的是：

```text
把人的做事方式，从“我推动工具”，切换成“系统孵化系统”。
```

## 03.4 四组件：元AI的装配清单

原力OS最终要造出来的东西，可以理解为一个元AI。

它不是一个聊天机器人，而是四套东西：

```text
文脉：灵魂
大脑：记忆
地图：判断
链路：手脚
```

### 文脉：灵魂

文脉回答“我是谁”。

它来自《原力资产》的我的说明书、母体定义卡、长期愿景、边界、意义主权。

没有文脉，AI 就不知道你是谁，只能给你通用答案。

### 大脑：记忆

大脑回答“我知道什么”。

它包括领域知识、案例、术语、判断、经验、资料库、客户记录和可检索知识库。

没有大脑，系统每次都从零开始。

### 地图：判断

地图回答“我怎么判断”。

它包括因果图、战略地图、商业模式、关键战役、指标体系、决策座舱。

没有地图，系统会有材料但不会判断。

### 链路：手脚

链路回答“我怎么行动”。

它包括 OSA、SOP、prompt、agent、workflow、自动化任务和交付流水线。

没有链路，系统只能想，不能做。

四套齐，才是一个懂你、记得住、会判断、能动手的工作生命体。

## 03.5 三法印：OS 的宪法

原力OS必须遵守三法印。

```text
递归进化
技能统帅
人类友好
```

它们不是普通原则，而是硅基世界的物理约束。

### 递归进化

每一件事做完，都必须留下“进化记忆”。

如果只剩下“完成了”，这件事价值是 1 倍。

如果留下“我比上次更知道怎么做”，这件事价值是 N 倍。

如果留下“AI 下次能直接调用这次经验”，这件事开始进入 ʸx。

### 技能统帅

接到任务，第一动作不是动手做，而是先搜索：有没有已经存在的 skill？

```text
有 → 复用。
没有 → 做之前先想，做完能不能蒸馏成 skill。
```

不 search 直接 call，就是重复造轮子。

### 人类友好

人类友好包括三件事：

```text
节省 token。
不打扰。
保留人的最终判断权。
```

它是三法印的根。

因为人是嵌套链的启动者。人崩塌，系统也会崩塌。

三法印优先级是：

```text
人类友好 > 递归进化 > 技能统帅
```

## 03.6 OSA：任务流的最小节拍

很多项目失败，不是因为执行力差，而是因为跳层。

人有一个本能：想到就做。

但“想到就做”经常从目标直接跳到行动，中间缺了策略。

原力OS用 OSA 把任务流拆成三段：

```text
O 目标层：我们要干什么？为什么是这件不是那件？
S 策略层：怎么打？哪条路径成功率最高？候选 ABC 是什么？
A 行动层：谁做？何时做？怎么验真？
```

任何任务必须先有完整 O，才能进入 S；先有完整 S，才能进入 A。

OSA 是嵌套链上的最小节拍。

它让任务可以被裁决、被拆解、被分发、被验真、被回写。

---

## 03.7 信息五层：让产出可蒸馏

一份输出好不好，不只看它是不是完整，而要看它停在哪一层。

```text
碎片 → 特征 → 观点 → 洞察 → 框架
```

层 1 是碎片，层 2 是标签，层 3 是观点，层 4 是洞察，层 5 是框架。

层 2 的输出，每次重做都要花同样的时间，是一次性消耗品。

层 4 的输出，可以蒸馏成 skill，下次直接调用。

所以信息五层不是审美问题，而是数学问题。

它决定你的产出是 x·y，还是 ʸx。

## 03.8 班底第一性：从一个人到 1+N

一个人独自工作，和带 AI 班底工作，不是同一个物种。

原力OS的基本工作形态是：

```text
1 + N + N 自繁殖
```

其中：

```text
1 = 你，碳基启动者
N = 你的 AI 班底
N 自繁殖 = AI 班底能造下一代 AI 班底 / skill / agent
```

如果 N 是固定的，你还在 x·y。

如果 N 在生长，你开始进入 ʸx。

最小可行班底可以从 3 个角色开始：

```text
策略大脑：负责复杂判断、候选方案、战略分析。
执行大脑：负责结构化执行、写代码、整理表格。
知识大脑：负责检索、校验、调用现成 skill。
```

## 03.9 蒸馏：case → pattern → skill

一次成功如果只用一次，就是一次性消耗。

一次成功如果能复用一百次，就是组织资产。

差别在蒸馏。

```text
case：一次性的事件
↓ 抽象
pattern：重复出现的模式
↓ 工程化
skill：可被反复调用的能力
```

蒸馏的难点不是写出 skill 文件，而是找准边界。

边界太窄，只能用一次。

边界太宽，用到不该用的地方，效果变差。

一个 skill 的成熟度可以分四级：

```text
v1 雏形：能跑通，自己用
v2 跑通：1-2 个客户成功用过，同事可试
v3 通用化：5+ 客户成功用过，进入 skill 库
v4 进库：全员可用，有官方文档和迭代机制
```

## 03.10 S路径：从学生到孵化者

原力OS的成熟路径叫 S 路径。

它不是能力清单，而是版本号。

```text
S1 建模阶段 = 1·x = 学生
S2 工具阶段 = x·y = 工匠
S3 板阶段   = x^y = 产品人
S4 舱阶段   = ʸx  = 孵化者
```

每一步都是相变，不是加法。

S1，你能听懂指令，完成基础任务。

S2，你能蒸馏 skill，能带 AI 班底。

S3，你能做出被深度依赖的产品。

S4，你能孵化下一个管培。

版本号让进化可量化。

没有版本号，你以为自己在长，其实可能没动。

## 03.12 本章边界

《原力OS》只负责三件事：

```text
把文脉、大脑、地图、链路造出来。
把每一次任务递归回写进去。
把 case 蒸馏成 pattern，再工程化成 skill。
```

它不负责重新找到你，那是《原力资产》。

它不负责判断借哪股势、做哪门生意，那是《原力创业》。

它负责把资产与创业产生的经验，变成可复用、可调用、可进化的系统。

本章的终点，是一句话：

> 我不再只是一个 AI 使用者，而是一个 ʸx 孵化者。

从这里开始，三部曲第一次闭环：

```text
找到你 → 借势你 → 变现你 → 系统你 → 复利你
```
