# Codex Task Template · v0.2

> 给 Codex 的任务必须是可执行、可验收、可回写的。不要把模糊愿望交给 Codex。  
> 母法：Codex 不是执行一次性劳动，而是把任务产物沉淀成下一次可复用资产。

---

## 1. 任务标题

```text
[归属 / 模块] 一句话说明任务
```

示例：

```text
[OS / C4] 从最小闭环样章提炼 Skill 蒸馏卡模板
```

## 5. 去重三缝检查

```text
本任务是否触碰“觉醒 / 找你 / 天才区”？是 / 否
若是：是否归 A 轴做深，B/C 只引用？

本任务是否触碰“觉醒 vs 变现”？是 / 否
若是：是否明确 A 止于显化，B 做深变现？

本任务是否触碰“元 AI”？是 / 否
若是：是否明确自用元 AI 归资产，业务班底归 OS？
```

## 10. 递归回写

```text

```

---

## 11. 禁止事项

```text
不要修改母公理。
不要新增未经裁决的核心概念。
不要把四部与四组件混成同一层。
不要引入无来源的文件结构。
不要生成无法维护的一次性代码。
不要把正文、模板、术语、接口混成同一文件。
不要绕过 project/backwrite-log.md。
```

---

## 12. Codex 可直接执行提示词

```text
请阅读本任务的背景、归属裁决、输入材料、去重三缝检查和验收标准。

执行前先输出：
1. 你理解的任务边界
2. 你识别的所属模块
3. 你不会修改的正典文件
4. 你计划生成或更新的文件

执行过程中不得修改母公理、三部边界和核心公式。
若发现任务触碰去重三缝，必须先按 interfaces/去重三缝.md 的规则处理。

完成后请输出：
1. 修改文件清单
2. 核心实现说明
3. 验收结果
4. 回写位置
5. 仍需人工裁决的问题
6. 建议下一步任务
```

```text

```

并同步：

