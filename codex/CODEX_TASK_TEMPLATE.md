# Codex Task Template

> 给 Codex 的任务必须是可执行、可验收、可回写的。不要把模糊愿望交给 Codex。

---

## 1. 任务标题

```text
[归属] 一句话说明任务
```

示例：

```text
[OS] 搭建文脉-大脑-地图-链路四组件管理页面
```

---

## 2. 背景

```text
这个任务来自哪里？
属于原力资产 / 原力创业 / 原力OS / 跨三部？
服务哪条共享脊骨？
为什么现在要做？
```

---

## 3. 第一性目标

```text
本任务不是为了 ____，而是为了 ____。
它最终要让下一次任务更容易，因为 ____。
```

---

## 4. 输入材料

```text
相关仓库：
相关文件：
相关概念：
已有设计：
不可改动的正典：
```

---

## 5. 输出物

```text
必须生成：
可选生成：
不得生成：
```

---

## 6. 验收标准

```text
1. 功能验收：
2. 内容验收：
3. 结构验收：
4. 风格验收：
5. 回写验收：
```

---

## 7. 递归回写

```text
完成后，哪些内容需要回写到文脉？
哪些内容需要进入大脑？
哪些内容更新地图？
哪些内容沉淀为链路或 SOP？
```

---

## 8. 禁止事项

```text
不要修改母公理
不要新增未经裁决的概念
不要把四部与四组件混成同一层
不要引入无来源的文件结构
不要生成无法维护的一次性代码
```

---

## 9. Codex 可直接执行提示词

```text
请阅读本任务的背景、输入材料和验收标准。
先列出你理解的任务边界，再执行。
执行过程中不得修改母公理、三部边界和核心公式。
完成后请输出：
1. 修改文件清单
2. 核心实现说明
3. 验收结果
4. 仍需人工裁决的问题
5. 建议回写位置
```
