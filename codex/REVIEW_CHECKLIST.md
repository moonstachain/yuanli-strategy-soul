# Review Checklist

Use this checklist before closing any v1.0 whiteboard OS task.

## 1. Output

- [ ] Output exists.
- [ ] Output matches the issue.
- [ ] No unrelated files changed.

## 2. Canon

- [ ] Mother axiom unchanged.
- [ ] Three-part structure unchanged.
- [ ] Twelve-module structure unchanged.
- [ ] Any canon change is clearly marked for governance.

## 3. Function

- [ ] Page or file can be opened/read.
- [ ] Navigation is consistent where relevant.
- [ ] User can find the next action.

## 4. Style

- [ ] Dark cockpit style.
- [ ] Warm gold emphasis.
- [ ] Dense but readable panels.
- [ ] No decorative AI-gradient dominance.

## 5. Backwrite

- [ ] Evolution Note drafted.
- [ ] Backwrite target named.
- [ ] Next reuse path named.
- [ ] Follow-up issue created if needed.

## Decision

- [ ] Done
- [ ] Review
- [ ] Backwrite needed
- [ ] Needs human decision
