# 收口句模板 · v0.1

> Issue: #206  
> 归属：templates/os / C4 一条链路  
> 来源：`yuanli-strategy-trilogy/books/03-原力OS/最小闭环.md`  
> 用途：把含糊的“先这样吧 / 回头再说 / 在推进中”，替换成可收束、可交接、可回写的一句话。

---

## 0. 母句

```text
这一步我做完了，结果是 ______，接下来该 ______ 做 ______。
```

这句话压缩了闭环四段：

```text
做完了 = 收束
结果是 X = 明确结果态
该谁做什么 = 推进到下一棒
```

---

## 1. 使用前提

当你准备说：

```text
先这样吧。
回头再说。
差不多了。
还在推进。
我再看看。
应该可以了。
```

请改用收口句。

---

## 2. 标准格式

```text
这一步我做完了，结果是【明确结果】，接下来该【下一棒】做【下一动作】。
```

### 字段说明

| 字段 | 要求 | 禁止 |
|---|---|---|
| 这一步 | 必须明确是哪一步 | 禁止泛泛说“这个事情” |
| 我做完了 | 必须真实完成当前动作 | 禁止用来掩盖未完成 |
| 结果是 | 必须有结果态或证据 | 禁止“还行、差不多、应该可以” |
| 接下来该谁 | 必须指定人或系统 | 禁止“大家、我们、回头” |
| 做什么 | 必须有下一动作 | 禁止“继续推进”这种空话 |

---

## 3. 三种结果态模板

### 3.1 completed · 完成

```text
这一步我做完了，结果是【文件/交付物/验证结果已经完成】，接下来该【谁】做【下一步动作】。
```

示例：

```text
这一步我做完了，结果是 A1 模块文件已经创建并写入来源、输入、输出和回写位置，接下来该 P1 任务继续提炼 A1 的工具卡。
```

---

### 3.2 blocked · 卡住

```text
这一步我做完了当前能做的部分，结果是【卡在什么地方】，接下来该【谁】提供【缺失条件】。
```

示例：

```text
这一步我做完了当前能做的部分，结果是无法确认某个术语的正典来源，接下来该 source-index 补齐原始出处。
```

---

### 3.3 failed_partial · 部分失败

```text
这一步我做完了验证，结果是【哪部分不成立 / 不通过】，接下来该【谁】做【修正动作】。
```

示例：

```text
这一步我做完了验证，结果是 B2 的心理账户定义与旧版“心智账户”存在混用，接下来该 glossary 统一正名并在接口文件中写明弃用规则。
```

## 4. 反模式替换表

| 含糊说法 | 为什么不合格 | 替换成 |
|---|---|---|
| 先这样吧 | 没有结果态、没有下一棒 | 这一步我做完了，结果是 ___，接下来该 ___ 做 ___。 |
| 回头再看 | 没有时间、没有责任人 | 这一步目前卡在 ___，接下来该 ___ 在 ___ 前确认 ___。 |
| 还在推进 | 没说明推进到哪 | 这一步已完成 ___，还缺 ___，接下来该 ___ 做 ___。 |
| 差不多了 | 无法验收 | 这一步未完成，因为 ___；完成标准是 ___。 |
| 我再研究下 | 没有输出物 | 这一步下一输出是 ___，预计由 ___ 产出。 |
| 大家看下 | 没有决策请求 | 我需要 ___ 对 ___ 做 yes/no 判断，判断后进入 ___。 |

## 5. 用于 GitHub Issue 的收口句

```text
本轮任务已完成【具体输出物】，结果是【文件路径 / commit / 验收结果】，接下来进入【下一阶段任务】。
```

示例：

```text
本轮任务已完成 A1-C4 十二模块文件，结果是 modules/ 下已有 12 个 v0.1 正典模块，接下来进入 glossary 与 interfaces 的 P1 吸收任务。
```

---

## 6. 用于 Codex 任务的收口句

```text
这一步我做完了，结果是【修改了哪些文件 / 生成了哪些文件 / 通过了哪些检查】，接下来该【人 / Codex / 下一个 agent】做【下一步】。
```

必须包含：

```text
文件路径
输出物
验收结果
回写位置
下一步
```

---

## 7. 用于团队沟通的收口句

```text
这一步我做完了，结果是【你现在可以看到 / 使用 / 判断的东西】，接下来该【某人】做【一个明确动作】。
```

---

## 8. 用于客户交付的收口句

```text
本阶段已完成【交付物】，核心结论是【一句话结论】，下一步建议【动作】，需要你确认【yes/no 选项】。
```

---

## 9. 自检

发送任何收口句前，检查：

```text
是否有明确结果？
是否有下一棒？
是否有下一动作？
是否没有“差不多、回头、推进中”？
是否能让对方立即接住？
```

五项不过，不许发。
