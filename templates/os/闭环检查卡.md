> 用途：把任何一件“还在推进中”的任务，收成一个可验收、可交接、可回写的最小闭环。

---

## 0. 使用场景

当一件事长期处于：

```text
还在弄
在推进中
差不多了
回头再说
先这样吧
```

就使用本卡。

本卡的目的不是让事情变复杂，而是把一个模糊任务收成：

```text
goal → action → evidence → writeback → mirror
```

## 3. 四段收口压缩版

如果任务太小，不必完整填写五段，可用四段压缩检查：

```text
① 连续表达：我把这件事讲/做到了一个完整段落吗？
② 收束：我明确说出“这一步做完了”了吗？
③ 推进：下一棒是谁，接什么？
④ 结果态：这一圈的结论是什么？
```

| 四段 | 截断条件 | 是否打钩 |
|---|---|---|
| 连续表达 | 意思已闭合，不是半句、半事、半交付 | □ |
| 收束 | 已明文收口，不是“先这样吧” | □ |
| 推进 | 下一棒清晰，人或系统知道接什么 | □ |
| 结果态 | 不含“待定、回头看、差不多” | □ |

---

## 4. Done 标准

本卡只有一个 Done 标准：

```text
五段全部闭合，或四段压缩版全部打钩。
```

如果任一段未闭合：

```text
不得报完成。
不得进入下一个任务。
不得写“已推进”。
必须说明卡在哪一段。
```
