## 0. Purpose

```text
USER-LANGUAGE-TEST-v1 tests whether target users can repeat, feel, and budget-shift around the new category names.
```

This test is not asking:

```text
Which name do we like best?
```

It is asking:

```text
Which name can the target user understand, repeat, feel pulled by, and justify paying for?
```

After higher-tier extraction, this protocol also tests:

```text
Which phrase should play which role in the naming stack?
```

The naming stack roles are:

```text
1. Public category name: user-repeatable, clear, externally usable.
2. Premium positioning: creates status, budget pull, and high-end meaning.
3. Bridge / explanation phrase: makes the category easier to understand.
4. Deep narrative line: preserves Yuanli taste and founder essence.
```

## 4. Test Flow

### Step 1 · Blind name exposure

Show only the name, not the explanation.

Ask:

```text
你第一眼觉得这是什么？
你觉得这是卖什么的？
这是给谁的？
你会把它归到哪个旧品类里？
```

### Step 2 · Two-sentence exposure

Show the two-sentence expression from `REAL-CASE-TEST-v1.md`.

Ask:

```text
现在你是否更清楚它是什么？
你能不能用自己的话复述一遍？
你觉得它和旧品类有什么不同？
```

### Step 3 · Role classification

Show all candidate phrases for the same case together.

Ask:

```text
如果这些词都可以留下：
哪个最适合放在封面标题？
哪个最适合做副标题或高端定位？
哪个最适合销售解释？
哪个更像一句品牌故事或广告语？
```

Role options:

```text
主品类名
高端定位语
销售解释语
深层叙事语
不建议使用
```

### Step 4 · Emotional pull

Ask:

```text
这个表达有没有让你心动、好奇、被击中、想进一步了解？
哪个词最打动你？哪个词让你出戏？
```

### Step 5 · Budget shift

Ask:

```text
如果你是目标用户，这个表达是否让你愿意重新分配预算或注意力？
你愿意为它支付更高溢价吗？为什么？
```

### Step 6 · Repeatability check

Wait 3-5 minutes, then ask:

```text
不看原文，你还记得这个东西叫什么吗？
你会怎么介绍给朋友？
```

---

## 5. Scoring Rubric

Score each name 1-5:

| Dimension | Question |
|---|---|
| First-glance clarity | Can the user guess what it means before explanation? |
| Target-user fit | Does the user know who it is for? |
| Old-category escape | Does it escape the old category? |
| Emotional pull | Does it create desire, curiosity, or identity resonance? |
| Repeatability | Can the user repeat it after a delay? |
| Budget pull | Does it justify paying more or reallocating attention? |
| Premium feel | Does it feel high-end enough for the target segment? |
| Yuanli fit | Does it preserve the strategic / aesthetic Yuanli taste? |
| Role clarity | Is it clear whether this phrase should be category, positioning, bridge, or narrative? |

Pass threshold:

```text
Average >= 4.0
Repeatability >= 4
Budget pull >= 4
No dimension below 3
```

Revise threshold:

```text
Average 3.2-3.9
Or repeatability / budget pull below 4
```

No-go threshold:

```text
Average < 3.2
Or user consistently misclassifies it as old category
```

## 9. Interview Script

```text
我会给你看几个名字和一句解释。你不用照顾我，也不用说好听的。请只回答你真实的第一反应。

第一轮：只看名字。
1. 你觉得这是什么？
2. 你觉得这是给谁的？
3. 你会把它归到什么旧品类？
4. 这个名字让你想进一步了解吗？

第二轮：看两句话解释。
1. 现在你是否更清楚？
2. 你能不能用自己的话复述？
3. 哪个词最打动你？
4. 哪个词让你出戏？
5. 你觉得它凭什么更贵？
6. 如果你要推荐给朋友，你会怎么说？

第三轮：角色归类。
如果这些词都可以留下：
1. 哪个最适合做封面标题 / 主品类名？
2. 哪个最适合做副标题 / 高端定位？
3. 哪个最适合做销售解释？
4. 哪个更像一句品牌故事或广告语？
5. 哪个不建议使用？

第四轮：3-5 分钟后复述。
1. 不看原文，你还记得它叫什么吗？
2. 你会怎么描述它？
3. 你会愿意继续了解、转介绍或付费吗？
```
