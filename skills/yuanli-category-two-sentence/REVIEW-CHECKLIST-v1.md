## 2. Scoring Scale

```text
1 = weak / unclear / likely misleading
2 = usable seed but strategically soft
3 = acceptable, needs revision before real test
4 = strong, can enter real case test with minor edits
5 = excellent, high strategic sharpness and immediate test value
```

---

## 3. Review Dimensions

| Dimension | Question |
|---|---|
| Target User Clarity | Is the target user specific enough? |
| Expensive Problem | Is the problem painful enough to justify attention or budget shift? |
| Old Category Escape | Does it clearly escape an old category/comparison trap? |
| New Category Claim | Does it create a memorable and defensible new frame? |
| Unique Mechanism | Does it explain why this is different, not just better? |
| Evidence Link | Is there a proof signal or source pattern? |
| Repeatability | Can a user repeat the core idea after hearing it once? |
| Budget Pull | Does it imply a reason to reallocate budget, time, or attention? |
| Yuanli Taste | Does it feel like Yuanli: strategic, compressed, elegant, non-generic? |
| Testability | Can it be tested with users next week? |

---

## 4. Decision Rule

### 4.1 Go to real case test

```text
Average score >= 4.0
No dimension below 3
No privacy issue
Clear next test exists
Ming approves
```

### 4.2 Revise before test

```text
Average score 3.2-3.9
Or one dimension below 3
Or old/new category is not sharp enough
```

### 4.3 No-go

```text
Average score < 3.2
Or target user / expensive problem is unclear
Or the expression is only slogan/copywriting
Or private data is involved
```

## 9. Cross-Sample Findings

### 9.1 What works

```text
1. 三个样本都成功逃离旧品类。
2. 三个样本都能说清楚目标用户和贵问题。
3. 三个样本都有明确的新分类倾向。
4. 三个样本都能进入真实用户测试。
```

### 9.2 What is still weak

```text
1. 证据语言还弱，更多是战略推断，不是用户原话证明。
2. 新品类命名仍偏工具化，如“工作台”“报告”“仪表盘”。
3. 原力审美还可以更强：需要更有结构张力、更像高端战略语言。
4. 还没有明哥人工品味审查。
```

### 9.3 Shared revision direction

```text
从“工具名”升级为“结果机制名”。
从“AI 提效”升级为“战略判断 / 可见性 / 决策证据”。
从“我们帮你做 X”升级为“我们把旧状态压缩/转译/升级成新系统”。
```
