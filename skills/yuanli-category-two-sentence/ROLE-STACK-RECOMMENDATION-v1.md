## 0. Executive Summary

```text
Role Stack Recommendation v1 converts Ming's first-round user feedback into recommended naming stacks for the three selected cases.
```

Core insight:

```text
用户语言比战略语言更具体。
```

Therefore, the recommendation is not to force one winning phrase. It is to assign each phrase to the role where users are most likely to understand, repeat, feel, and pay.

## 1. Role Stack Definitions

```yaml
public_category:
  meaning: first-glance, user-repeatable, externally usable category phrase
premium_positioning:
  meaning: status / budget-pull phrase that supports high-end meaning
business_explanation:
  meaning: phrase that makes the business category credible after context/story/proof
bridge_explanation:
  meaning: user-friendly sales explanation that connects category to felt value
deep_narrative:
  meaning: Yuanli / founder-level emotional or philosophical narrative
community_narrative:
  meaning: phrase that helps private-domain community understand identity and behavior
product_mechanism:
  meaning: concrete mechanism explaining why product is different
diagnostic_internal_phrase:
  meaning: strategically accurate phrase for internal use, not necessarily user-facing
plain_language_anchor:
  meaning: clear control phrase that may lack originality but checks comprehension
```

# 2. CASE-01 · 水月老师随身寺庙

## 2.1 Field Signal

```yaml
blind_name_signal: 用户不听故事、只看名字时，仍然比较容易理解。
after_story_signal: 听完故事、出处之后，用户愿意把它理解为精神资产。
signal_strength: medium_positive
```

## 2.2 Recommendation

```yaml
public_category: 随身寺庙
business_explanation: 随身精神资产
bridge_explanation: 可带走的东方精神器物
deep_narrative: 有出处、有敬意、有能量的东方精神器物
not_primary:
  - 随身精神资产 as first public title
```

## 2.3 Recommended Two-Layer Public Expression

```text
随身寺庙
一件可带走的东方精神器物，听完出处之后，它不只是文玩，而是有精神来源的随身精神资产。
```

## 2.4 Reasoning

```text
“随身寺庙”通过了初步理解门槛，适合作为主品类入口。
“随身精神资产”在故事、出处、材料和证明之后更成立，适合作为商业解释。
“可带走的东方精神器物”更像销售桥接语，帮助用户把诗性品类落到可理解对象上。
```

## 2.5 Remaining Validation Need

```text
1. 3-5 分钟后用户是否还能复述“随身寺庙”。
2. “随身寺庙”与“随身精神资产”谁更容易被转介绍。
3. 听完出处后，用户愿意多付钱的强度评分。
```

# 3. CASE-05 · 达哥张姐低奢皮草

## 3.1 Field Signal

```yaml
gift_to_close_friend_signal: 暂时还不会自然想到送闺蜜。
repeat_purchase_signal: 轻奢 / 轻度奢侈方向能让用户想到复购。
high_net_worth_friend_signal: 送给高净值朋友时，会更像高净值姐姐的东西。
signal_strength: mixed_positive
```

## 3.2 Recommendation

```yaml
public_category_candidate: 养成系稀缺美学
plain_language_anchor: 高净值姐姐专属高端皮草
budget_context: 轻奢复购 / 高净值朋友赠礼
community_narrative_candidate: 养成系社交货币
product_mechanism_candidate: 双奢结合
phrase_to_deprioritize:
  - 闺蜜礼物 as primary gift scene
```

## 3.3 Recommended Two-Layer Public Expression

```text
养成系稀缺美学
为高净值姐姐打造的轻奢复购型高端皮草，把一件皮草从普通服饰，变成小圈层里拿得出手的高净值朋友礼物。
```

Alternative clearer version:

```text
高净值姐姐专属高端皮草
用张姐的个人审美和稀缺材质，把轻奢皮草做成可复购、可转介绍、可送给高净值朋友的小圈层身份表达。
```

## 3.4 Reasoning

```text
“闺蜜礼物”暂时没有自然触发，应降级为次级假设。
“轻奢/轻度奢侈”更接近当前用户对复购的理解。
“高净值朋友”比“闺蜜”更贴近当前礼物场景。
“高净值姐姐专属高端皮草”虽然不够独创，但有很强的明白话锚点价值。
“养成系稀缺美学”仍适合作为更有审美和品类感的主候选，但需要更多验证。
```

## 3.5 Remaining Validation Need

```text
1. 用户是否能自然复述“养成系稀缺美学”。
2. “轻奢复购”是否应该进入公开话术。
3. “高净值姐姐专属高端皮草”是否虽然直白，但转化效率更高。
4. “双奢结合”是否只是机制解释，不应作为品类名。
```

# 4. CASE-08 · 高净值留学生游戏化婚礼

## 4.1 Field Signal

```yaml
story_seen_budget_signal: “自己的故事被看见”特别被认可，用户愿意为此付费。
gamification_signal: 游戏化是年轻人喜欢的，并且特别适合朋友推荐。
life_narrative_signal: 人生叙事未必特别好理解。
signal_strength: strong_positive
```

## 4.2 Recommendation

```yaml
public_category: 游戏化社交婚礼
premium_positioning_candidate: 归国首秀导演
bridge_explanation: 让自己的故事被看见
deep_narrative: 人生主权宣言
diagnostic_internal_phrase: 人生叙事的空间翻译
phrase_to_deprioritize_as_public: 人生叙事的空间翻译
```

## 4.3 Recommended Two-Layer Public Expression

```text
游戏化社交婚礼
为高净值归国年轻人设计一场让自己的故事被看见的社交首秀，让婚礼不只是布置，而是朋友愿意参与和转发的人生主权宣言。
```

Alternative premium-positioning version:

```text
归国首秀导演
把婚礼变成高净值归国年轻人的游戏化社交首秀，让自己的故事被看见，也让家人和朋友看到一个真正长大的你。
```

## 4.4 Reasoning

```text
“游戏化”得到年轻用户喜欢，并且适合朋友推荐，因此适合作为外部传播入口。
“自己的故事被看见”是当前最强预算牵引语，应进入桥接解释。
“人生叙事的空间翻译”战略上准确，但用户不够好理解，应降级为内部诊断语。
“归国首秀导演”仍可能承载高端定位，但这次反馈没有直接解决，需要后续补问。
```

## 4.5 Remaining Validation Need

```text
1. “游戏化社交婚礼”是否能在 3-5 分钟后被复述。
2. “归国首秀导演”是否能支撑高端定位和溢价。
3. “让自己的故事被看见”是否应进入标题或副标题。
4. “人生叙事的空间翻译”是否只保留为内部诊断语。
```
