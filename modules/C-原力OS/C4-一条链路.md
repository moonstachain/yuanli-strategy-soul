## 一句话定义

一条链路，是把战略地图中的判断，转化为可重复执行、可交给人或 AI、可验收、可回写、可自动化的行动流程。

## 不是它

```text
不是待办事项。
不是临时任务。
不是一次性交付。
不是没有验收标准的流程。
不是“我做了”但没有 evidence 的动作。
不是不能沉淀为 skill 的消耗型劳动。
```

## 第一性问题

```text
如何把战略判断变成可复用、可执行、可自动化的行动？
```

## 原子模块

```text
C4.1 OSA 任务链：目标、策略、行动。
C4.2 最小闭环：goal → action → evidence → writeback → mirror。
C4.3 SOP 链：固定流程与验收标准。
C4.4 Agent 链：可交给 AI 执行的任务流。
C4.5 Codex 链：输入、输出、文件、验收、禁止事项、回写位置。
C4.6 内容链：选题、生产、分发、复盘。
C4.7 交付链：客户、方案、执行、验收、复购。
C4.8 回写链：任务结束后写回文脉、大脑、地图、链路与母体。
```

## 输入

```text
C3 一张地图
任务目标
工具能力
模板
验收标准
历史流程
项目看板
Codex-ready task
```

## 判断标准

```text
是否能重复执行？
是否能交给别人或 AI？
是否有明确输入输出？
是否有验收标准？
是否有 evidence？
是否有回写位置？
是否能沉淀为 SOP / skill / workflow？
```
